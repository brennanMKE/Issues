## Description

The four view-mode segments in `StatsBarViewModeSwitcherView` (Swimlanes / Timeline / List / Recent) have no `.help(...)` modifier, so hovering reveals nothing. The view-mode keyboard shortcuts (Cmd+Opt+1..4 by default, customizable in Settings) are discoverable only via the View menu. Adding a tooltip with the shortcut on each segment is a tiny self-teaching addition.

## Implementation sketch

Each segment Button gets:

```swift
.help(helpText(for: mode))
```

where the helper renders the mode's display name plus the formatted shortcut from `AppCommandsController.shared.shortcuts.binding(for:)`. Use `ShortcutBinding.displayString` (the existing pretty-printer that produces `⌘⌥1`).

```swift
private func helpText(for mode: IssueStore.ViewMode) -> String {
    let action: ShortcutAction = switch mode {
    case .swimlane: .swimlanesView
    case .timeline: .timelineView
    case .list: .listView
    case .recent: .recentView
    }
    let glyph = AppCommandsController.shared.shortcuts.binding(for: action).displayString
    return "\(mode.displayName) (\(glyph))"
}
```

The binding the user picks in Settings will reflect in the tooltip — no manual sync needed.

## Other places to consider (not in scope here)

- Status filter pills already have `.help(...)`; leave alone.
- The `+` button in the tab bar has `.help("Open another folder in a new tab")`; could append `(⌘T)` from the configurable binding.
- Reload, Close Tab, etc. — same pattern, but those live in menu items rather than visible buttons, so the tooltip lift is less helpful.

A follow-up could sweep tooltips across everything, but the view-mode segments are the most visible self-teaching opportunity.

## Verification

- Hover the "Swimlanes" segment → tooltip appears: `Swimlanes (⌘⌥1)` (or whatever the user has bound).
- Change the binding in Settings → Shortcuts → tooltip reflects the new combo without restart.
- All four segments show their respective shortcut.

## Root cause

The four segment buttons in `StatsBarViewModeSwitcherView` had no `.help(...)` modifier, so the configurable view-mode shortcuts (default `⌘⌥1…4`) were only discoverable through the View menu.

## Fix

Added a `.help(helpText(for: mode))` modifier to each segment button. The helper maps `IssueStore.ViewMode` to the matching `ShortcutAction`, asks `AppCommandsController.shared.shortcuts.binding(for:)` for the user's current binding, and formats the tooltip as `"<displayName> (<glyph>)"` using the existing `ShortcutBinding.displayString` pretty-printer. Because the lookup happens each time SwiftUI re-evaluates the view, rebinding the shortcut in Settings → Shortcuts is reflected immediately without a restart.

## Files changed

- `Issues/Views/StatsBarViewModeSwitcherView.swift` — added `.help(...)` modifier and the private `helpText(for:)` helper.

## Gotchas

- `AppCommandsController.shared` is `@MainActor`, which is fine here — the view itself runs on the main actor, so the lookup is synchronous and free.
- Used the `switch` expression form (Swift 5.9+) for the action mapping; the project's deployment target (macOS 26.4) easily supports it.
