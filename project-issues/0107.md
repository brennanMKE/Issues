# 0107 — Debug DMG build script for ad-hoc testing on the remote Mac

| | |
|---|---|
| **Status** | open |
| **Module** | Build |
| **Platform** | macOS |
| **First seen** | 2026-05-08 |

## Description

Phase 5 of the remote-access plan (#0076). The release flow (`scripts/release.sh`) produces a signed/notarized DMG suitable for general distribution. For iterating on remote-access changes, that loop is unnecessarily slow — we want a way to drop a Debug build (with full os_log output, asserts on, no notarization round-trip) onto the test Mac in seconds, run it once, and throw it away.

This issue adds a parallel `scripts/debug-dmg.sh` that produces a DMG of the Debug configuration, ad-hoc signed (or simply unsigned with Gatekeeper override via `xattr -d com.apple.quarantine`).

## Where to look

- **`scripts/release.sh`** — the existing release script, used as the structural model. Don't modify it.
- **New: `scripts/debug-dmg.sh`** — a sibling script that:
  - Builds the Debug config: `xcodebuild -project Issues.xcodeproj -scheme Issues -configuration Debug -destination 'platform=macOS' -derivedDataPath build/debug build`.
  - Locates the produced `Issues.app` under `build/debug/Build/Products/Debug/`.
  - Either ad-hoc signs (`codesign --force --sign -`) or skips signing entirely.
  - Wraps it in a DMG via `hdiutil create -volname "Issues Debug" -srcfolder Issues.app -ov -format UDZO Issues-debug.dmg`.
  - Prints a one-liner the user can run on the destination Mac after copying:
    ```
    xattr -d com.apple.quarantine /Applications/Issues.app
    ```

## Behavior

- The script is idempotent — running twice produces a fresh DMG, overwriting any prior `Issues-debug.dmg`.
- Output path: `build/Issues-debug.dmg` so it's easy to `scp` from the build Mac.
- Exit nonzero on any build / sign / dmg-create step failure.
- Optional `--clean` flag wipes `build/debug/` first.
- Debug builds keep `ENABLE_TESTABILITY = YES` and full DWARF, so `os_log` filtering with `--level debug` works on the destination Mac.

## Verification

- `scripts/debug-dmg.sh` produces `build/Issues-debug.dmg`.
- Copy the DMG to the test Mac, mount, drag to `/Applications`, run the `xattr -d` command, launch — Issues.app starts. (The `xattr -d` is needed for ad-hoc-signed apps to bypass Gatekeeper.)
- `log stream --predicate 'subsystem == "co.sstools.Issues"' --level debug` on the test Mac shows the full debug output.
- Test the remote-access flow against this Debug build → connects, serves, etc.

## Notes

- Keep this script lightweight. It's not for distribution; it's for "I want to test this on my Mac mini in the next 60 seconds."
- The `xattr -d com.apple.quarantine` step is only required because the build is ad-hoc signed. If we ever want to skip it, we'd need to sign with the Developer ID identity, which slows the loop down enough to defeat the script's purpose.
- This script is **not** a substitute for `scripts/release.sh`. Production use stays on the signed/notarized path.

## Scope (out)

- Auto-deploying to a known remote Mac via `scp` or a watch loop. Could be a small follow-up if the manual copy gets tiresome.
- Building for arm64 vs. x86_64 separately. Default arch is fine.
