# 0044 — Add Quick Look-style spacebar preview to open the markdown sheet

| | |
|---|---|
| **Status** | in-progress |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |

## Description

After #0043 dropped double-click open in List view (because the count: 2 gesture racing NSTableView's selection couldn't be made reliable), the user-facing affordances to open the markdown sheet from List are: Enter on a selected row, right-click → Preview Markdown, or the detail-panel link. None of those carry the discoverability of macOS's Quick Look pattern — Finder, Mail, Photos, Music all open a preview when the user hits the spacebar on a selected item.

Adding spacebar as a universal "preview the selected issue" gesture sidesteps the Table gesture problem entirely (it's keyboard, not mouse) and brings us in line with macOS conventions across every view mode at once.

## Fix

In `Issues/Views/MainView.swift`, `contentArea` already has an `.onKeyPress(.return)` that opens `markdownSheetIssue`. Add a parallel `.onKeyPress(.space)` with the same body:

```swift
.onKeyPress(.space) {
    if let issue = store.selectedIssue {
        markdownSheetIssue = issue
        return .handled
    }
    return .ignored
}
```

The handler attaches to the same `.focusable()` that owns the existing arrow-key navigation, so:
- Selecting a row in List view (single-click), Recent, Swimlane, Timeline → focus is on `contentArea` → spacebar fires.
- When the search field is focused (`StatsBarSearchField`'s `@FocusState`), spacebar types a space in the field as expected — the keypress doesn't bubble to `contentArea` because focus is elsewhere.
- When the markdown sheet is open, focus moves into the sheet; the contentArea handler doesn't fire. Esc still dismisses (the sheet's existing `.keyboardShortcut(.cancelAction)`).

## Out of scope

- Quick Look's "press space again to close" toggle. Closing requires the modal sheet to capture spacebar, which means another keyPress handler on `IssueMarkdownSheet` itself. Worth considering, but ship the open path first; users who want to close still have Esc and the close button.
- True `QLPreviewPanel` integration. The markdown sheet uses Textual's `StructuredText` for rich rendering; swapping to `QLPreviewPanel` would lose that and only give us the file's plain markdown source. Not a win.

## Verification

- Select any row in List view via single-click. Press space → markdown sheet opens.
- Same in Recent, Swimlane, Timeline.
- Use ↑/↓ to move selection. Press space → sheet opens for the now-selected row.
- Click into the search field, type a phrase that includes spaces → spaces type normally, sheet does not open.
- With the sheet open, press Esc → sheet closes.
