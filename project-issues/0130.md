## Description

Translate every key in `Localizable.xcstrings` to Korean (`ko`). Depends on `#0059` being complete (the catalog and the `ko` slot must exist before content can land); follows the same pattern as `#0060` / `#0061` / `#0062`.

## Translation guidance

- Locale code is **`ko`** (region-neutral; covers both `ko-KR` and the rare `ko-KP`).
- Tone: macOS Korean localization uses the **하십시오체** (formal-imperative) register for app chrome — short, polite, impersonal. Avoid 반말 (informal) entirely. Apple's localizations consistently use this register.
- Match Apple's Korean conventions where possible:
  - "Close" → "닫기"
  - "Reload" → "다시 불러오기"
  - "Settings" → "설정"
  - "Open" (file/folder) → "열기"
  - "Cancel" → "취소"
  - "Done" → "완료"
- Status display names:
  - "Open" → "미해결"
  - "In Progress" → "진행 중"
  - "Resolved" → "해결됨"
  - "Closed" → "닫힘"
  - "Won't Fix" → "수정 안 함"
- View modes:
  - "Swimlanes" → "스윔레인"
  - "Timeline" → "타임라인"
  - "List" → "목록"
  - "Recent" → "최근"
- Plural-aware strings (`^[\(count) issue](inflect: true)`): Korean **doesn't inflect for plural** — the same noun form works for any count. Set the `other` inflection variant only (single slot for `ko`).

## Particles and spacing

Korean uses **postpositional particles** that attach to nouns and follow vowel/consonant rules (`이/가`, `을/를`, `은/는`, `와/과`, `(으)로`). When string interpolation injects a noun and a particle follows, Xcode's catalog supports the `^[…](particle: true)` morphology variant — use it for any sentence template where a particle follows an interpolated value.

Spacing: Korean uses single-byte ASCII spaces between word boundaries. Don't use full-width spaces. Korean **does** typically space around the inflection point (e.g. "진행 중", not "진행중"), matching Apple's convention.

Punctuation: half-width ASCII (`.` `,` `?` `!` `:`) is the macOS norm in Korean — not the full-width forms. Match Apple's own apps.

## Process

1. Open `Issues/Localizable.xcstrings` in Xcode.
2. For each key, fill in the `ko` slot.
3. Build with the system language set to Korean (`AppleLanguages = ("ko")` in the scheme's environment, or change the system preference) and walk every screen.
4. Pay attention to controls with fixed widths (status pills, view-mode capsule, the `+` button) — Korean text is typically 5–15% **longer** than English (often single Hangul syllables compose into wider visual blocks). The tight spots are status pills and view-mode capsules.

## Verification

- Run with system language `ko` → every visible English string renders in Korean.
- No truncation in the toolbar, stats bar, settings panes.
- Pluralization renders correctly (1개의 문제 / 2개의 문제 — confirm the single-form inflection rule is set).
- Particle morphology works: any string of the form "{folder name}을(를) 열까요?" renders as "메모를 열까요?" (vowel-ending) or "프로젝트를 열까요?" (consonant-ending) without showing the parenthetical alternative.
- Switching back to English flips cleanly (no caching of the previous locale).

## Out of scope

- Translating issue markdown bodies (user content).
- Translating the IssuesSkill format spec.
- Adapting date formats — Apple's `DateFormatter` already adapts via the active locale; just confirm dates render Korean-style (`2026년 5월 12일`).
- North Korean (`ko-KP`) variants. Region-neutral `ko` covers the practical user base.

## Notes

- The Korean catalog format compiles into a `ko.lproj` directory at build time; the SwiftUI runtime picks it up based on the user's preferred language list.
- Korean has a strong stylistic preference for keeping app-chrome strings short — verbose translations stand out as awkward. When in doubt, prefer the more concise Apple-app translation.
- Technical terms (e.g. "Markdown", "Quick Look", "WebSocket") are typically left in English in Korean localizations, sometimes with Hangul phonetic transliteration in parentheses on first use ("마크다운"). Keep them English unless Apple's own apps transliterate.
- For composite English+Korean strings (e.g. "Open `0042.md` in Quick Look"), wrap the English token in a code-style font / inline-code marker where possible to make the language switch visually obvious. The catalog supports inline formatting; check what Apple does in Mail or Notes for similar mixed strings.

## Fix

Filled the `ko` localization for every key in `Issues/Localizable.xcstrings` following the formal-imperative (하십시오체) register noted in the spec. Status names map "Open / In Progress / Resolved / Closed / Won't Fix" to "미해결 / 진행 중 / 해결됨 / 닫힘 / 수정 안 함"; view modes map "Swimlanes / Timeline / List / Recent" to "스윔레인 / 타임라인 / 목록 / 최근". Half-width ASCII punctuation per Apple convention. Disambiguation: "Revoke" → "무효화" (invalidate, distinct from "Cancel" → "취소"); "Rotate" / "Rotate certificate…" → "갱신" / "인증서 갱신…" (renewal, not physical rotation) — both follow Apple's security-context conventions.

No particle-bearing string templates appeared in v1 (no `%@`-injected nouns followed by `이/가` / `을/를` / `(으)로`), so the catalog's particle-morphology variant wasn't needed. The future "Trying again — since %@" string uses `%@부터` which composes correctly with any time-stamp formatting Apple's `DateFormatter` returns.

## Gotchas

- **"Revoke" ≠ "Cancel"** in Korean. Both map to "취소" in casual usage, but in the access-token context "Revoke" needs a stronger verb. Used "무효화" (invalidate) to keep them distinct in the UI.
- **"Rotate" in cert context ≠ physical rotation** ("회전"). Used "갱신" (renewal/refresh) per Apple's security-context conventions. The cert-rotation strings ("Rotate certificate…" / "Confirm certificate rotation" / "Rotation invalidates…") all use 갱신 consistently.
- **Half-width punctuation only.** Korean macOS conventions use ASCII `.` / `,` / `?` / `!` — not the full-width forms common in Chinese / Japanese. Don't switch to full-width just because the surrounding text is CJK.
- **Korean visual width runs ~5–15% longer than English.** Status pills and the view-mode capsule are the tight spots; current text fits but a future contributor adding a longer string should check the toolbar before shipping.
- **Format-string `%@` is preserved.** `"다시 시도 중 — %@부터"` keeps the `%@` token followed by the particle `부터`. The composition works for any timestamp the runtime injects.
