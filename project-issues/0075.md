# 0075 — Read project.json from each watched folder and use its name for the tab title

| | |
|---|---|
| **Status** | open |
| **Module** | Models / State |
| **Platform** | macOS |
| **First seen** | 2026-05-08 |

## Description

The IssuesSkill now writes a small `project.json` at the root of every issues folder it manages — capturing the project's display name and repo URL (see `IssuesSkill/references/project-config.md`). Issues.app should consume this file so tabs show the project's chosen display name instead of the parent folder name.

This unblocks the remote-access plan (see `RemoteAccess.md`) — the folder picker on a remote viewer needs each folder's `project.json` `name` to render meaningful labels — and it's a quality-of-life win locally too: a tab labeled "Issues.app" reads better than `Issues`, and "Bluesky for SwiftUI" reads better than `bluesky-swiftui`.

## Schema (consumer side)

Issues.app reads the file lenient-style — anything it doesn't recognize is ignored, missing fields fall back to today's behavior. The fields it cares about for v1:

```json
{
  "name": "Issues.app",
  "url": "https://github.com/brennanMKE/Issues"
}
```

- **`name`** — the tab's display label. Replaces the parent-folder fallback when present.
- **`url`** — repo URL. Decoded into the model now (single read pass) but not surfaced in the UI yet; future issues will add an "Open repository" link in the detail panel and the remote folder picker subtitle.

The schema authority is the IssuesSkill, not this app. New fields in `project.json` should be ignored by Issues.app until support is explicitly added.

## Behavior

- **Has `project.json` with `name`** → tab title is `name`. The detail panel and any breadcrumb/header that today shows the parent folder name uses `name` instead.
- **No `project.json`** → today's behavior: tab title is the parent folder name.
- **`project.json` exists but `name` is empty / missing / null** → fall back to parent folder name, same as if the file were absent.
- **`project.json` exists but is malformed** (invalid JSON, wrong type) → log a warning through the existing `co.sstools.Issues` logger and treat as if missing. Don't surface a parse error in the UI; `project.json` is optional metadata.
- **`project.json` is added/edited/removed at runtime** → the existing FSEvents debounce already triggers `IssueStore.reload()` on any change inside the watched folder. The reload pass should re-read `project.json` so a fresh edit takes effect within ~150 ms, the same as any other file change.

## Where to look

- **`Issues/Models/`** — new `ProjectMetadata.swift` with a `Decodable` struct (`name: String?`, `url: URL?`). Use `URL?` so a missing or empty url decodes cleanly to nil.
- **`Issues/State/IssueStore.swift`** — owns folder loading. Add a `projectMetadata: ProjectMetadata?` field on the store; populate it during `reload()` by reading `<root>/project.json` (one extra small file read per reload, well under the parser's per-issue cost). Don't pre-cache between reloads — the FSEvents pass already coalesces save bursts.
- **Tab title source** — find via `grep -rn "displayName\|tabTitle\|folderName" Issues/State/ Issues/Views/`. The current rendering likely walks back to the parent URL's `lastPathComponent`. Replace with `projectMetadata?.name ?? <parent folder name>`.
- **Tests** — `IssuesTests/`. Add a test class (or extend the existing one) covering: file present + valid `name`; file present with empty/missing `name`; file present but malformed JSON; file absent. The fallback path is the easy case to break and should have a regression test.

## Verification

- Open this repo's `project-issues/` folder in Issues.app — tab should now show **Issues.app** (the value just committed in `project-issues/project.json`) instead of **project-issues**.
- Open another folder that has no `project.json` — tab still shows the parent folder name.
- Edit `project-issues/project.json` to change `name` — tab updates within ~1 sec via FSEvents.
- Make the JSON malformed (e.g. delete the closing brace) — tab falls back to parent folder name; `log stream --predicate 'subsystem == "co.sstools.Issues"'` shows a warning, no crash.
- Delete `project.json` — tab falls back to parent folder name on next reload.

## Notes

- This is the first consumer of `project.json` in Issues.app, but the data shape is small and stable enough that adding it now (rather than waiting for the remote-access work) is low risk and immediately useful.
- The `url` field decodes into the model now even though no UI surfaces it yet — that way the follow-up "Open repository" issue is just a view change, not another data-layer pass.
- `URL?` decoding with an empty string gives nil naturally on `URL(string:)`-style decoders; for `Decodable` synthesis, declare the property as `URL?` and let `JSONDecoder` handle missing/null. If decoding strictness causes problems with empty-string `url` values, decode as `String?` and convert.
- This is **not** a regression and shouldn't change anything for folders without `project.json`. The fallback path is the existing default, just made explicit.
- Scope intentionally excludes UI surfacing of `url`, the remote folder picker, and per-folder hosting toggles — those are tracked separately in the `RemoteAccess.md` plan.
