## Description

Translate every key in `Localizable.xcstrings` to Simplified Chinese (Mandarin, `zh-Hans`). Depends on `#0059` being complete (the catalog and the `zh-Hans` slot must exist before content can land); follows the same pattern as `#0060` / `#0061` / `#0062`.

## Translation guidance

- Locale code is **`zh-Hans`** (Simplified, region-neutral). Don't use `zh-CN` — Apple's catalog format keys on script (`Hans`/`Hant`) rather than region for macOS.
- Tone: macOS Simplified Chinese localization is **terse and impersonal**, matching Apple's own apps. Avoid "您" / "你" except where direct address is unavoidable; prefer imperative or noun-phrase forms ("打开文件夹…" not "请你打开文件夹…").
- Match Apple's Simplified Chinese conventions where possible:
  - "Close" → "关闭"
  - "Reload" → "重新载入"
  - "Settings" → "设置"
  - "Open" (file/folder) → "打开"
  - "Cancel" → "取消"
  - "Done" → "完成"
- Status display names:
  - "Open" → "未处理"
  - "In Progress" → "处理中"
  - "Resolved" → "已解决"
  - "Closed" → "已关闭"
  - "Won't Fix" → "不予修复"
- View modes:
  - "Swimlanes" → "泳道图"
  - "Timeline" → "时间线"
  - "List" → "列表"
  - "Recent" → "最近"
- Plural-aware strings (`^[\(count) issue](inflect: true)`): Chinese **doesn't inflect for plural** — the same form works for any count. Set the `other` inflection variant only (Xcode's catalog editor surfaces it as a single slot for `zh-Hans`).

## Punctuation

Simplified Chinese uses **full-width** punctuation in most app contexts. The catalog editor accepts these directly via input method or paste:

- Period: `。` (not `.`)
- Comma: `，` (not `,`)
- Colon: `：` (not `:`)
- Question mark: `？` (not `?`)
- Exclamation: `！` (not `!`)
- Quotation: `「…」` for traditional contexts, `"…"` (full-width) for modern UI strings — match Apple's convention, which is full-width straight quotes.

Numbers and Latin proper nouns (e.g. URLs, file extensions like `.md`) stay half-width.

## Process

1. Open `Issues/Localizable.xcstrings` in Xcode.
2. For each key, fill in the `zh-Hans` slot.
3. Build with the system language set to Simplified Chinese (`AppleLanguages = ("zh-Hans")` in the scheme's environment, or change the system preference) and walk every screen.
4. Pay attention to controls with fixed widths (status pills, view-mode capsule, the `+` button) — Chinese text is typically 30–40% **shorter** than English on character count but each character is wider, so visual width works out close to English. Worth double-checking the tightest spots (stats bar, toolbar).

## Verification

- Run with system language `zh-Hans` → every visible English string renders in Simplified Chinese.
- No truncation in the toolbar, stats bar, settings panes.
- Pluralization renders correctly (1 个问题 / 2 个问题 — confirm the single-form inflection rule is set).
- Switching back to English flips cleanly (no caching of the previous locale).
- Punctuation is full-width in narrative strings; half-width in tokens (URLs, file extensions).

## Out of scope

- Traditional Chinese (`zh-Hant`). Separate ticket if/when the user requests it.
- Translating issue markdown bodies (user content).
- Translating the IssuesSkill format spec.
- Adapting date formats — Apple's `DateFormatter` already adapts via the active locale; just confirm dates render Chinese-style (`2026年5月12日`).

## Notes

- The Chinese catalog format compiles into a `zh-Hans.lproj` directory at build time; the SwiftUI runtime picks it up based on the user's preferred language list, not the system region. A user in the US with their preferred language set to "Chinese, Simplified" gets this localization.
- Avoid mechanical machine translation if possible — Apple's own Chinese localizations are a stronger reference than DeepL/Google output for app-chrome strings. When in doubt, pull up the same control in System Settings or Finder under Simplified Chinese and match.
- Some technical terms (e.g. "Markdown", "Quick Look") are typically left in English even in Chinese localizations. Keep them English unless Apple's own apps translate them.
