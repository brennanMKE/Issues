# 0091 — Manual host:port fallback field on viewer

| | |
|---|---|
| **Status** | open |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-08 |

## Description

Phase 2 of the remote-access plan (#0076). Bonjour discovery (#0089) doesn't traverse every network — Tailscale without MagicDNS, coffee-shop Wi-Fi with client isolation, etc. The viewer needs a typed-in `host:port` field as the fallback so the user can connect even when discovery is silent.

## Where to look

- **`Issues/Views/RemoteFolderPickerView.swift`** (created in #0097) — adds the manual-entry row.
- **State plumbing into the eventual remote tab type** (#0099) — a manually-entered host should be persisted per-tab so reconnects don't re-prompt.

## UI

```
┌─ Connect to remote host ───────────────────────────┐
│                                                    │
│  Discovered hosts:                                  │
│    • Brennan's MacBook Air     · 3 folders          │
│    • Brennan's Mac mini        · 1 folder           │
│                                                    │
│  ─ or enter manually ─                              │
│                                                    │
│  Host:    [ 100.74.12.5             ]              │
│  Port:    [ 51823                   ]              │
│                                                    │
│                          [ Cancel ] [ Continue ]    │
└────────────────────────────────────────────────────┘
```

- Two text fields. Host accepts an IPv4, IPv6, or DNS name (Tailscale MagicDNS names like `mac-mini.tail-scale.ts.net` work as-is). Port is integer 1–65535.
- "Continue" is disabled until both fields are valid.
- Single combined field (`host:port`) was considered and rejected — splitting catches mistakes in the port faster.

## Behavior

- On Continue, the viewer attempts `GET /v1/host` against `<host>:<port>` with no token (just to check connectivity). 401 means "reachable, awaiting token" → proceed to the token paste step (#0096). Connection refused, timeout, or DNS fail → inline error under the field, no advance.
- Connect timeout is 3 seconds (the user is sitting in front of this; longer feels broken).
- The pair `(host, port)` is persisted as a `RememberedHost` keyed by an internal id (a hash of `host:port`). The picker shows previously-used manual hosts in a "Recent" list so the user doesn't retype on every connect.
- Manual hosts and Bonjour-discovered hosts share the same downstream code path once a connection is established — they both just produce a `(host, port, token)` triple.

## Verification

- With Bonjour disabled (e.g. on a network where mDNS doesn't traverse), enter a known-good `host:port` → reaches the token entry step.
- Wrong port → "couldn't reach this host" inline error within 3 seconds.
- IPv6 link-local (`fe80::abcd…%en0`) — works if the user knows the right interface zone. We don't try to be clever about this; `Network.framework` parses it.
- Tailscale name (e.g. `mac-mini`) works when MagicDNS is on.
- After successful connect, the host appears in the "Recent" list on next launch with the same `(host, port)`.

## Notes

- Don't pre-resolve DNS in the field. `Network.framework` does it on connect; failing fast is fine.
- We're showing both a discovered list **and** a manual-entry section in the same picker because users on partial-mDNS networks need both at hand. A toggle to hide one or the other isn't worth the UI complexity for v1.
- The "Recent" section is purely a convenience and can be cleared per-host with a context menu on the row.

## Scope (out)

- Token paste UI — #0096.
- Per-host Keychain helpers — #0095.
- Network-change pause for the viewer (e.g. drop a manual connection when leaving Tailscale) — out of scope; reconnect logic in #0103 handles it implicitly.
