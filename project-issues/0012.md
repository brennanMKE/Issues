# 0012 — Stale-bookmark error state with Locate folder action

| | |
|---|---|
| **Status** | open |
| **Module** | Services / Views |
| **Platform** | macOS |
| **First seen** | 2026-05-03 |

## Description

When a persisted tab's security-scoped bookmark fails to resolve at launch (folder moved, renamed, or deleted), the tab is silently dropped. Instead, keep the tab in the list and render it in an error state with a "Locate folder…" button that opens an `NSOpenPanel` to re-bind the bookmark.

## Notes

- `TabsModel.restore()` is where bookmarks resolve. Today an unresolvable bookmark is appended to a "drop" list and pruned. Replace with a `TabState.unresolved(displayPath: String, bookmarkData: Data)` case.
- `RootView` / `MainView` renders the empty/error placeholder when the active tab is unresolved.
- "Locate folder…" reuses `FolderBookmarkService.presentOpenPanel()` and then re-creates the bookmark from the resolved URL.
- `FolderBookmarkService` already handles stale bookmarks for folders that resolve; this adds the case where resolve outright fails.
- PRD calls this out as nice-to-have for #0002; this is the dedicated follow-up.
