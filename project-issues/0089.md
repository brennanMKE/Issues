## Description

Phase 2 of the remote-access plan (#0076). The viewer side of Bonjour: discover hosts publishing `_issues._tcp.local.` (#0088) and expose them as a published list the picker UI can bind to. Cross-platform — the iPhone target will reuse this file unchanged when #0108 lands.

## Where to look

- **New: `Issues/Remote/RemoteServiceBrowser.swift`** — `NWBrowser` wrapper that exposes a `@Published` (or `@Observable`) `[DiscoveredHost]`.
- **`Issues/Info.plist`** — `NSBonjourServices` entry already added in #0093 lets the OS allow the browse.

## API

```swift
@Observable
@MainActor
final class RemoteServiceBrowser {
    private(set) var hosts: [DiscoveredHost] = []
    func start()   // begins NWBrowser, populates `hosts`
    func stop()    // cancels the browser; clears `hosts`
}

struct DiscoveredHost: Identifiable, Hashable {
    let id: String              // Bonjour service name (stable while service is published)
    let displayName: String     // from TXT record `n`, falls back to id
    let folderCount: Int?       // from TXT record `c`
    let endpoint: NWEndpoint    // for connecting
}
```

- The browser is `start`/`stop`able so we can avoid running it when the user isn't on a discovery screen.
- `hosts` is sorted by `displayName` ascending for deterministic UI rendering.
- A host is removed from the list when its Bonjour record disappears (the host toggled off, network change, etc.).

## Behavior

- Use `NWBrowser(for: .bonjourWithTXTRecord(type: "_issues._tcp", domain: nil), using: …)`.
- For each `NWBrowser.Result` parse the TXT record into the `displayName` / `folderCount` fields. Missing TXT (older host? bug?) → use the service name as `displayName`, `folderCount = nil`.
- The browser doesn't connect — it only enumerates. Connection happens later in the viewer flow (#0095, #0099).
- On iOS, the first invocation of `start()` triggers the local-network permission prompt (controlled by `NSLocalNetworkUsageDescription` in #0093). Handle the deny case: if permission is denied, `hosts` stays empty and we surface "manual entry" guidance through the picker.

## Verification

- Run two Macs on the same Wi-Fi. Host A toggles on → Host B's browser sees it within 2 seconds.
- Host A toggles off → disappears from Host B's `hosts` within a couple of seconds.
- TXT updates (rename host, change folder count) — Host B's row updates in place; `id` stays the same.
- Bring up Tailscale on both Macs with MagicDNS — Host A still appears (because Tailscale forwards mDNS in that mode). Without MagicDNS, Host A doesn't appear; the manual `host:port` field in #0091 is the path.

## Notes

- Don't try to merge / dedupe by IP. Bonjour service-name is the right identity; if the user has two Macs publishing the same service name, mDNS will already have appended ` (2)` (per #0088), so both rows show up legitimately.
- Avoid logging discovered host details verbosely on every browse update — gets noisy fast on a busy network.
- Cross-platform: this file should compile against both macOS and iOS without `#if`. The only iOS-specific concern is the permission prompt, which is OS-driven.
