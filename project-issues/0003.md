# 0003 — Show a change indicator on inactive tabs when their folder updates

| | |
|---|---|
| **Status** | open |
| **Module** | Views / State |
| **Platform** | macOS |
| **First seen** | 2026-05-03 |

## Description

Once the tab UI from #0002 lands, inactive tabs should display a visual indicator (e.g. a colored dot on the tab chip) when their watched folder receives changes — issue files added, removed, or modified — so the user knows there's something new to look at without switching tabs to check. Activating a tab clears its indicator.

## Expected behavior

- Each tab tracks an "unseen changes" boolean. The indicator is hidden by default.
- When a background tab's `IssueStore` reloads in response to a `FolderWatcher` event and the resulting `[Issue]` array differs from what the user last saw on that tab, the indicator turns on.
- The active tab never shows the indicator — switching to a tab marks it seen and clears the indicator.
- The indicator is a small dot (4–6pt) tinted with the app accent color (or a status-tinted color if the change involved a status transition; accent is fine for v1).
- Initial folder load on tab creation does **not** trigger the indicator — the user just opened the tab, so they're implicitly caught up.

## Notes

- "Last seen state" can be a snapshot of issue ids + their `Status` + file mtime, kept on the `IssueStore` (or on the `TabsModel`'s tab struct) at the moment the tab last became active. Compare against the new `[Issue]` after each `reload()`.
- A simpler heuristic — "any reload while inactive flips the bit" — would also work and is cheaper, but produces false positives when a save doesn't actually change anything the user cares about. Try the snapshot diff first; fall back to the simple flag if the diff cost is annoying.
- The indicator should also fire on tab restore at launch if the folder's contents differ from a persisted snapshot (nice-to-have; defer if it complicates persistence).
- Accessibility: include the change state in the tab's `.help(...)` tooltip and `.accessibilityLabel(...)` ("Updated since last viewed").
- Depends on #0002. Do not start until the tab UI exists.
