## Description

Phase 3 of the remote-access plan (#0076). The picker UI (#0097) is one half — the other is remembering what the user picked so reconnecting doesn't re-prompt, and giving them a way to add/remove folders later without going through the connect-a-host flow again.

## Where to look

- **`Issues/Views/RemoteFolderPickerView.swift`** (from #0097) — same view, used both in the connect flow and the manage flow.
- **New: `Issues/Views/ManageSubscriptionsSheet.swift`** — thin wrapper around the picker that adds/removes remote tabs (#0099) for an already-connected host.
- **New: `Issues/State/RemoteSubscriptionStore.swift`** — `UserDefaults`-backed persistence keyed by `hostId`.
- **`Issues/Models/TabPersistedState.swift`** — already exists; extend with the per-tab `(hostId, folderId, tokenStored: Bool)` triple.

## State shape

```swift
struct RemoteSubscription: Codable, Identifiable {
    let hostId: String              // Bonjour name or hash(host:port)
    let hostDisplayName: String
    var folderIds: [String]         // currently subscribed folders for this host
    var lastConnectedAt: Date
}

@Observable
final class RemoteSubscriptionStore {
    private(set) var subscriptions: [RemoteSubscription] = []
    func setFolders(_ ids: [String], forHost id: String, displayName: String)
    func removeFolder(_ id: String, forHost hostId: String)
    func forgetHost(_ hostId: String)
    func subscription(for hostId: String) -> RemoteSubscription?
}
```

Persisted as JSON in `UserDefaults` under `RemoteSubscriptions.v1`. Tokens stay in the Keychain (#0095); only the *what's subscribed* metadata lives here.

## Behavior

- **First connect to a host** → picker shows nothing pre-selected. Confirm writes the selection via `setFolders`.
- **Reconnect to a known host** (token already in Keychain, picker reopened later) → previously-selected folders are pre-checked; new folders show up unchecked.
- **A previously-selected folder no longer exists on the host** (host removed it from the share list, or `folderId` changed via #0082) — drop it silently from the persisted list. Don't surface a "this is gone" alert; the user notices when the manage sheet shows fewer rows.
- **Forget host** removes both the subscription record here and the token in `ViewerTokenStore` (#0095). Surfaced as a button in the Manage Subscriptions sheet, not in the picker proper.

## Manage Subscriptions sheet

Reachable via:

- A menu item under the existing tab/folder picker entry (e.g. "Manage Remote Subscriptions…").
- A button in the host settings on a connected tab's overflow menu.

The sheet shows:

- One section per known host (with a header showing its display name + "Forget host" button).
- Inside each section, the same row UI as #0097 — checkbox + name + description + repo link.
- Confirm button: "Apply changes". Compares the new selection to the persisted one and:
  - For newly-checked folders → opens new remote tabs (#0099).
  - For newly-unchecked folders → closes those tabs (the WS subscription is dropped automatically when the tab closes per #0102).

If the host isn't currently reachable, show its rows greyed out with a "Disconnected" badge; the user can still uncheck a folder (it'll apply on next reconnect).

## Verification

- Connect to a host, select two folders, Confirm. Two tabs open. Quit and relaunch. The same two tabs reopen; reconnecting to the host doesn't re-show the picker.
- Open Manage Subscriptions, uncheck one folder, Apply. That tab closes. The other stays.
- Add a third folder via the manage sheet. New tab opens.
- "Forget host" — token removed from Keychain (#0095), subscription removed, all that host's tabs close.
- Host removes a folder from its share list (#0085) — viewer's picker drops it from the persisted record on next refresh; no error UI.

## Notes

- Persisting only `folderId` (not display name) means the picker always renders fresh names from `/v1/folders`. If the user renamed a project's `project.json`, the new name shows up immediately.
- A "show only previously selected" toggle was considered for the manage sheet; rejected as one extra control for a small win. The list will be short.

## Scope (out)

- Auto-reconnect logic on launch — #0103 (uses this store as input).
- Find-by-name fallback when `folderId` changes — partly here (silent drop), partly in #0103 (re-resolve on reconnect).
- Per-folder remembered filters — the local app doesn't have this either; out of scope.
