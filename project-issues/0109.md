# 0109 — Use Quick Look for screenshot attachments instead of the custom modal sheet

| | |
|---|---|
| **Status** | open |
| **Module** | Views |
| **Platform** | macOS |

## Description

Clicking an inline image attachment in the detail panel currently opens `AttachmentSheet` — a custom modal window with its own zoom buttons, pan handling, and Reveal-in-Finder. Clicking a video attachment opens the system Quick Look panel via `.quickLookPreview($url)` (added in #0073). Quick Look gives us native pan / pinch-zoom / arrow-key navigation / Space-bar dismiss / "open with" / share — everything the custom sheet reimplements, plus the affordances it doesn't.

Now that the video path proves Quick Look is the right metaphor here, screenshots should use the same panel. The custom `AttachmentSheet` then becomes dead code and can be deleted.

## Expected behavior

- Clicking an inline image thumbnail opens the system Quick Look panel — same panel Finder shows on Space-bar — with all standard affordances (pinch-zoom, arrow keys to dismiss, "open with", share menu, etc.).
- Dismissing Quick Look returns focus to the detail panel; subsequent clicks reopen Quick Look cleanly.
- Behaviour for video links (#0073) is unchanged — they were already on Quick Look.

## Actual behavior

- Clicking a screenshot opens `AttachmentSheet`, a custom modal with its own toolbar (filename, Reveal-in-Finder, close) and a `ScrollView`-based pan/zoom implementation. Functional but inconsistent with the video path and missing the system affordances.

## Where to look

- **`Issues/Views/DetailPanelDescriptionView.swift`** — holds two separate `@State` URLs (`attachmentURL` for the sheet, `quickLookURL` for Quick Look). Collapse to one URL routed through `.quickLookPreview($url)`.
- **`Issues/Views/AttachmentThumbnailView.swift`** — `handleClick()` calls `onOpen(resolvedURL)` for images and `onPlay(url)` for videos. Both should now flow through the same Quick Look callback. The `onOpen` / `onPlay` distinction can collapse if it isn't needed elsewhere.
- **`Issues/Views/AttachmentSheet.swift`** — entire file becomes dead code; delete it. There are no references outside `DetailPanelDescriptionView` (verified via grep).

## Notes

- Quick Look on macOS handles `.png` / `.jpg` / `.heic` / `.pdf` / `.log` / `.txt` etc. natively, so the same panel handles non-image attachments without us adding type-specific code. The previous "open with `NSWorkspace`" branch for `.pdf` link targets in `AttachmentThumbnailView.handleClick()` can also collapse into the Quick Look path.
- The existing `.quickLookPreview` import + binding in `DetailPanelDescriptionView` already takes care of macOS 13+ availability — this project targets macOS 15+ so no annotation needed.
- This is a small cleanup, not a regression — `AttachmentSheet` was correct for its time, but `.quickLookPreview` makes it redundant.
