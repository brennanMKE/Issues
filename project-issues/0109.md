## Description

Clicking an inline image attachment in the detail panel currently opens `AttachmentSheet` — a custom modal window with its own zoom buttons, pan handling, and Reveal-in-Finder. Clicking a video attachment opens the system Quick Look panel via `.quickLookPreview($url)` (added in #0073). Quick Look gives us native pan / pinch-zoom / arrow-key navigation / Space-bar dismiss / "open with" / share — everything the custom sheet reimplements, plus the affordances it doesn't.

Now that the video path proves Quick Look is the right metaphor here, screenshots should use the same panel. The custom `AttachmentSheet` then becomes dead code and can be deleted.

## Expected behavior

- Clicking an inline image thumbnail opens the system Quick Look panel — same panel Finder shows on Space-bar — with all standard affordances (pinch-zoom, arrow keys to dismiss, "open with", share menu, etc.).
- Dismissing Quick Look returns focus to the detail panel; subsequent clicks reopen Quick Look cleanly.
- Behaviour for video links (#0073) is unchanged — they were already on Quick Look.

## Actual behavior

- Clicking a screenshot opens `AttachmentSheet`, a custom modal with its own toolbar (filename, Reveal-in-Finder, close) and a `ScrollView`-based pan/zoom implementation. Functional but inconsistent with the video path and missing the system affordances.

## Where to look

- **`Issues/Views/DetailPanelDescriptionView.swift`** — holds two separate `@State` URLs (`attachmentURL` for the sheet, `quickLookURL` for Quick Look). Collapse to one URL routed through `.quickLookPreview($url)`.
- **`Issues/Views/AttachmentThumbnailView.swift`** — `handleClick()` calls `onOpen(resolvedURL)` for images and `onPlay(url)` for videos. Both should now flow through the same Quick Look callback. The `onOpen` / `onPlay` distinction can collapse if it isn't needed elsewhere.
- **`Issues/Views/AttachmentSheet.swift`** — entire file becomes dead code; delete it. There are no references outside `DetailPanelDescriptionView` (verified via grep).

## Notes

- Quick Look on macOS handles `.png` / `.jpg` / `.heic` / `.pdf` / `.log` / `.txt` etc. natively, so the same panel handles non-image attachments without us adding type-specific code. The previous "open with `NSWorkspace`" branch for `.pdf` link targets in `AttachmentThumbnailView.handleClick()` can also collapse into the Quick Look path.
- The existing `.quickLookPreview` import + binding in `DetailPanelDescriptionView` already takes care of macOS 13+ availability — this project targets macOS 15+ so no annotation needed.
- This is a small cleanup, not a regression — `AttachmentSheet` was correct for its time, but `.quickLookPreview` makes it redundant.

## Root cause

`AttachmentSheet` predated the Quick Look path that landed for videos in #0073. Once `.quickLookPreview($url)` was wired up for videos, the only remaining justification for the custom sheet was that nobody had switched images over to the same binding. Two state vars (`attachmentURL` driving `.sheet`, `quickLookURL` driving `.quickLookPreview`) coexisted in `DetailPanelDescriptionView` for the same kind of presentation.

## Fix

- `DetailPanelDescriptionView` now has one `@State var quickLookURL` instead of two. Both the image callback (`onOpenAttachment`) and the video callback (`onPlayVideo`) write to it; `.quickLookPreview($quickLookURL)` opens whichever URL was assigned. The `.sheet { AttachmentSheet(url:) }` block is gone.
- `AttachmentThumbnailView.handleClick()` keeps the same three branches but the non-video link branch now invokes `onOpen(url)` instead of `NSWorkspace.shared.open(url)` — that routes PDFs, logs, etc. through Quick Look too, which previews them natively.
- `Issues/Views/AttachmentSheet.swift` is deleted. Grep confirmed no references outside `DetailPanelDescriptionView` (which no longer references it either).
- Doc comments on `AttachmentThumbnailView` updated to describe the Quick Look behaviour rather than the old sheet.

## Files changed

- `Issues/Views/DetailPanelDescriptionView.swift` — drop `attachmentURL` state and the `.sheet` block; rewire `onOpenAttachment` to write to `quickLookURL`. Updated the doc comment on `quickLookURL` to note it's the single binding now.
- `Issues/Views/AttachmentThumbnailView.swift` — collapse the `NSWorkspace.shared.open` branch into `onOpen` so non-video link targets also flow through Quick Look. Doc comments rewritten to describe the Quick Look path.
- `Issues/Views/AttachmentSheet.swift` — deleted.

## Gotchas

- **SwiftUI's `.sheet` requires `.sheet(isPresented:content:)` or `.sheet(item:)` — there's no `.sheet(url:content:)`.** The original code used the `isPresented` form with a derived `Binding` that read `attachmentURL != nil`. Replacing it with `.quickLookPreview` is a straight swap because Quick Look accepts a `Binding<URL?>` directly; no derived binding needed.
- **Quick Look on macOS reuses one panel across invocations.** Setting `quickLookURL` twice in a row (e.g. clicking one screenshot, dismissing, clicking another) reuses the same panel. The two callbacks writing to the same binding compose correctly; SwiftUI dismisses-and-represents when the URL changes.
- **`.quickLookPreview` is declared in the `QuickLook` module on macOS** (the `import QuickLook` at the top of `DetailPanelDescriptionView` was already there from #0073). Don't drop that import while removing the sheet — without it the modifier still appears missing on `some View`.
