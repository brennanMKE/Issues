# 0042 — List view lost double-click-to-open after #0040 round 2

| | |
|---|---|
| **Status** | in-progress |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |

## Description

After #0040 round 2 stripped per-cell gestures from `ListView` and moved double-click handling to a Table-level `.onTapGesture(count: 2)`, double-clicking a row no longer opens the markdown sheet. Single-click row selection works correctly now (which was the goal of #0040), but the open-markdown affordance is gone in List view. Recent view's double-click still works because its rows are plain `HStack`s, not `Table` rows.

## Root cause

`Table` in SwiftUI on macOS is backed by `NSTableView`. NSTableView consumes mouse events at the AppKit layer for row selection and doesn't bubble them up to a SwiftUI gesture recognizer attached to the Table view. So `.onTapGesture(count: 2)` on the Table's outer modifier chain never fires from a row click — it would only fire on a click outside any row (e.g. column-header empty space, where there's no AppKit consumer).

## Fix

Re-attach the double-tap gesture at the **cell** level, but as a `simultaneousGesture` and **without** re-adding `.contentShape(Rectangle())` — that combination was the round-1 problem. The trick:

- `.simultaneousGesture` lets the gesture fire alongside NSTableView's row-selection rather than racing for exclusive ownership.
- Skipping `.contentShape(Rectangle())` keeps the SwiftUI gesture's hit region bounded to the actual rendered `Text` glyph. Clicks elsewhere in the cell (mostly empty `.frame(maxWidth: .infinity)` whitespace) have no SwiftUI gesture to compete with and go cleanly to NSTableView for selection.

The trade-off: double-click only registers when the user clicks on the visible text. Empty whitespace within a cell (e.g. trailing space in the Title column) won't trigger open. In practice users target visible content, so this is acceptable.

```swift
// Each TableColumn cell, e.g.:
TableColumn("#", value: \.id) { issue in
    Text("#\(issue.id)")
        .font(.system(size: 11, weight: .heavy))
        .foregroundStyle(Color.appMuted)
        .frame(maxWidth: .infinity, alignment: .leading)
        .simultaneousGesture(TapGesture(count: 2).onEnded { onOpenMarkdown(issue) })
}
```

The Table-level `.onTapGesture(count: 2)` added in #0040 round 2 should be removed — it never fires anyway.

The Table-level `.contextMenu(forSelectionType: String.self)` from #0040 round 2 stays — that one does work because right-click goes through NSTableView's context-menu pathway which SwiftUI's modifier hooks into.

## Verification

- Single-click anywhere on a row in List view → still selects reliably (must remain working — this is the #0040 win we don't want to regress).
- Double-click on a row's visible text in any column → markdown sheet opens.
- Right-click on any row → "Preview Markdown" context menu still appears.
- Cmd+click on a selected row → still deselects.
