## Description

Phase 3 of the remote-access plan (#0076). After the user picks a host (Bonjour-discovered or manually entered), they paste the bearer token. This is the UI for that step plus the validation round-trip that confirms the token works before the picker advances to folder selection (#0097).

## Where to look

- **New: `Issues/Views/RemoteTokenPasteView.swift`** — the paste sheet content.
- **`Issues/Views/RemoteFolderPickerView.swift`** (#0097) — embeds this view as a step in the picker flow.
- **`Issues/Remote/ViewerTokenStore.swift`** (#0095) — stores the token after a successful validate.
- **`Issues/Remote/RemoteClient.swift`** (#0094) — provides `validate(token:)` that hits `GET /v1/host` with the token.

## UI

```
┌─ Connect to Brennan's MacBook Air ─────────────────┐
│                                                    │
│  Paste an access token from the host:              │
│                                                    │
│  [ iat_                                          ]  │
│                                                    │
│  Generate one in Issues.app on the host:           │
│  Settings → Remote Access → Access tokens.         │
│                                                    │
│  [ ✓ Save this token in Keychain ]                 │
│                                                    │
│                          [ Cancel ] [ Continue ]    │
└────────────────────────────────────────────────────┘
```

- Single-line text field. Auto-trims whitespace on paste.
- "Save this token in Keychain" checkbox is on by default. If unchecked, the token is held in memory for this session only (the user gets re-prompted on relaunch).
- "Continue" is enabled when the field looks well-formed (`iat_` prefix + 43 base64url characters = 47 chars total).

## Behavior

- On Continue, call `RemoteClient.validate(token:)` which hits `/v1/host` with the token.
  - 200 → store in Keychain (if checkbox on), advance to the folder picker (#0097).
  - 401 → inline error: "This token wasn't accepted by the host." Don't advance, don't store.
  - Connection error → inline error with retry. Don't advance.
- Validation is best-effort: a 200 here doesn't guarantee the token won't expire later, only that it's valid right now.
- If the user already has a token cached for this host (matching `hostId`), the paste step is **skipped** — the picker goes straight to folder selection. If that cached token now returns 401, drop back to this paste step with an explanatory inline message ("the token was revoked or expired").

## Verification

- Paste a valid token → advances to folder picker.
- Paste a malformed token (e.g. drops the `iat_` prefix) → Continue is disabled.
- Paste a well-formed but invalid token → 401 inline error.
- Paste a valid token with checkbox off → connect works, relaunch the app and reconnect → re-prompted.
- Paste a valid token with checkbox on → reconnect on relaunch is silent.

## Notes

- The "Save in Keychain" toggle defaults on because that's the common case. It's exposed for the user who wants ephemeral access (e.g. on a borrowed Mac).
- Don't auto-paste from the clipboard unless the user explicitly clicks a "Paste from clipboard" button — auto-pasting tokens from system pasteboard has a clear privacy footprint.
- We considered a "scan QR" button here for parity with the iPhone flow. Out of v1 scope for the Mac viewer; the iPhone gets it in #0108.

## Scope (out)

- Folder picker — #0097.
- "Forget this host" UX — surfaces in #0098.
- QR scanning on Mac — out of v1.

## Resolution

Landed as Phase B of `RemoteFolderPickerView` in the bundled viewer-UI commit `4635e4b` (merge `acb3876`). The token field is a monospace `TextEditor` (multi-line, the brief's preference over the single-line `TextField` in the original mock) with Back / Cancel / Continue. Continue calls `URLSessionRemoteHostProbe.fetchHost(host:port:token:)`; 200 → store via `ViewerTokenStore.store(token:forHost:)` and advance to folder-pick; 401 → inline "Token rejected." The bearer never reaches `os_log` and only lives in the view's `@State` until Phase C confirms. See `project-issues/0097.md`'s resolution for the full writeup.
