## Description

Phase 1 of the remote-access plan (#0076). The host needs an in-process HTTP server that listens on the local network, validates bearer tokens against `AccessToken` (#0078), and dispatches to a small route table. This issue covers only the listener, the auth middleware, and the route plumbing — the actual endpoint handlers (folder enumeration, issue reading, attachment streaming) are #0080 and #0081, and the WebSocket upgrade is #0100.

The server lives inside `Issues.app` — there is no separate daemon, no LaunchAgent. Quitting the app stops serving (matches #0074).

## Where to look

- **New: `Issues/Remote/RemoteServer.swift`** — `NWListener` + a tiny route table. Gated `#if os(macOS)`.
- **No external dependency.** Use `Network.framework` (`NWListener`, `NWConnection`) directly. Don't pull in Vapor, swift-nio, or a third-party HTTP server — the route surface is small enough that a hand-rolled HTTP/1.1 parser over `NWConnection` is shorter and has no version-skew risk.
- **`Issues/State/IssueStore.swift`** — the server reads from the same store(s) the local app uses; no parallel data layer.

## Behavior

- **Bind**: by default listens on all non-loopback IPv4/IPv6 interfaces (per #0076 decision). A future issue can add interface pinning if needed.
- **Port**: configurable, default `0` (let the OS pick) on first run. Persist the chosen port to `UserDefaults` under `RemoteServer.port` so reconnects are stable across launches.
- **TLS**: not used in v1. The shared-secret auth + Tailscale's tunnel encryption (when Tailscale is the path) covers the threat model.
- **Concurrency**: each accepted `NWConnection` reads a single HTTP/1.1 request, dispatches, writes the response, and closes (or upgrades to WebSocket per #0100). Request handling is `Task { … }` per connection.
- **Auth middleware**: every request must carry `Authorization: Bearer iat_<…>`. Missing or unparseable header → `401 Unauthorized` with body `{"error":"missing_token"}`. Invalid token → `401 invalid_token`. Expired token → `401 expired_token`. Valid → request proceeds and `AccessToken.touch(hash:from:)` fires asynchronously so the read path isn't blocked on the keychain write.
- **Route table**: a simple `[Route]` matched by `(method, pathPattern)`. Patterns support `{folderId}` and `{id}` segment placeholders captured into a `[String: String]` for the handler. Unmatched routes → `404 not_found`.
- **Errors**: any thrown error from a handler maps to `500 internal_error` with the error description in the body in **debug builds only**; release builds return `500 internal_error` with no body.
- **CORS / preflight**: not handled. The viewer is a native client, not a browser.

## Verification

- Toggle hosting on. `lsof -nP -iTCP -sTCP:LISTEN | grep Issues` shows the listener on the chosen port.
- `curl http://<ip>:<port>/v1/host` → `401 missing_token`.
- `curl -H "Authorization: Bearer iat_garbage" http://<ip>:<port>/v1/host` → `401 invalid_token`.
- `curl -H "Authorization: Bearer iat_<valid>" http://<ip>:<port>/v1/host` → `200` (handler returns once #0080 lands; for this ticket alone, a placeholder handler returning `{ "ok": true }` is fine).
- Toggle hosting off → port no longer listening.
- Quit the app while a connection is in flight → connection closes cleanly, no crash.

## Notes

- Hand-rolling HTTP/1.1 is fine for `GET` requests with small bodies. Keep the parser strict and return `400 bad_request` on anything weird (oversize headers, missing `Host`, body on a `GET`).
- Use `os.Logger` on `co.sstools.Issues` subsystem with category `RemoteServer` so `log stream` works the way it does for the rest of the app.
- The auth middleware is the load-bearing security boundary. It's worth a tight unit test pass before any handler logic exists.

## Root cause

Greenfield. With #0078 (tokens) and #0086 (entitlements) in place, the host needed a working listener + auth pipeline so the per-endpoint issues (#0080, #0081, #0100) can each be a small route handler against an established harness.

## Fix

Single new file `Issues/Remote/RemoteServer.swift` deliberately structured so the security-critical parts are pure-function-testable and the `NWListener` wrap is thin on top:

- **HTTP types.** `HTTPRequest`, `HTTPResponse`, plus a hand-rolled HTTP/1.1 parser. Strict on purpose: non-`GET` → `unsupportedMethod`, body bytes after the header terminator → `bodyOnGet`, missing `Host` → `missingHost`, oversize headers (>16KB) → `headersTooLarge`. Path is query-stripped before matching; header lookups are case-insensitive (RFC 7230 §3.2). `HTTPResponse.serialized()` injects `Content-Length` and `Connection: close` and sorts header keys for stable wire output.
- **Route table.** `Route` rows hold `(method, pathPattern, handler)`. `RouteTable.match` segments both inputs on `/`, and any pattern segment of the form `{name}` captures into the dictionary handed to the handler. Method comparison is case-insensitive. Length mismatch is an immediate non-match — no greedy patterns, no globs, no regex.
- **Auth middleware.** `extractBearer` parses the `Authorization` header (case-insensitive `Bearer ` scheme per RFC 6750), then `authenticate` defers to `AccessToken.validate` (#0078). Outcomes are an enum so the dispatch table maps cleanly to the spec'd 401 reasons (`missing_token`, `invalid_token`, `expired_token`). The middleware does **not** call `touch` — the server's request loop schedules that on a detached task after the response is on the wire.
- **Listener wrap (`#if os(macOS)`).** `@MainActor final class RemoteServer` opens an `NWListener` (TCP, no TLS in v1, all-interface listen via the default `NWParameters.tcp`), persists the chosen port to `UserDefaults["RemoteServer.port"]`, and accepts connections on a global `userInitiated` queue. Each accepted `NWConnection` becomes a `Task` that reads up to 16KB or `\r\n\r\n`, parses, auths, routes, replies, and closes. State changes hop back to MainActor for `connections` / `connectedPeers` mutation.
- **Forward-declared `MultiFolderStore` protocol.** Required by the spec's `init(store: MultiFolderStore)`, satisfied with a one-method protocol (`hostedFolderIds: [String]`) that #0085 will extend. Avoids blocking #0079 on a type that doesn't exist yet, and keeps the route handlers decoupled from the concrete tab model.
- **Default routes (placeholder).** `/v1/host` returns `{"ok":true}`; `/v1/folders` returns `{"folders":[…]}` from `MultiFolderStore.hostedFolderIds`. Both get replaced by #0080's real handlers.

## Files changed

- `Issues/Remote/RemoteServer.swift` — new. Everything above. ~370 lines including doc comments.
- `IssuesTests/RemoteServerTests.swift` — new. 18 tests covering parser (minimal GET, query strip, non-GET reject, body-on-GET reject, missing-Host reject, case-insensitive Authorization), router (literal match, placeholder capture, mismatch on shape/length, case-insensitive method), auth (extractBearer shape coverage; missing/malformed/invalid/expired/valid via real `AccessToken` calls with isolated Keychain services), and `HTTPResponse` serialization shape.

## Gotchas

- **Don't be liberal in what the parser accepts.** The auth middleware sits one layer behind the parser; a "be permissive" parser is a security smell. The `400 bad_request` cases are not user-facing — only the viewer talks to this — so strict rejection is the correct default.
- **`@MainActor` listener + Network.framework callbacks.** `NWListener.stateUpdateHandler` and `newConnectionHandler` fire on the queue you pass to `start(queue:)`. The handlers here hop to `Task { @MainActor }` for any state mutation; don't be tempted to mutate `connections` / `connectedPeers` from the network queue directly.
- **Token `touch` off the response path.** A Keychain write per request would be unfortunate. Detached `Task { try AccessToken.touch(...) }` after `await send(...)` keeps the response cheap; if the touch fails it's logged but doesn't affect the response the viewer already received.
- **Persisted port may collide.** `UserDefaults["RemoteServer.port"]` carries the last bound port; if it's in use on next launch, `NWListener` fails and `handleListenerState` calls `stop()`. The user's recourse for now is "toggle hosting off and on" — `/v1/folders` clears the saved port on stop. If we see this in the wild, file a follow-up to clear the persisted port on `errSecAddrInUse`.
- **`MultiFolderStore` is a forward declaration.** The empty/single-method shape was chosen deliberately: filling it out belongs in #0085 with the actual per-folder hosting toggles. Resist the urge to grow it here just because we can.
