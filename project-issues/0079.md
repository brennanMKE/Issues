# 0079 — RemoteServer: HTTP listener with bearer auth and route table

## Description

Phase 1 of the remote-access plan (#0076). The host needs an in-process HTTP server that listens on the local network, validates bearer tokens against `AccessToken` (#0078), and dispatches to a small route table. This issue covers only the listener, the auth middleware, and the route plumbing — the actual endpoint handlers (folder enumeration, issue reading, attachment streaming) are #0080 and #0081, and the WebSocket upgrade is #0100.

The server lives inside `Issues.app` — there is no separate daemon, no LaunchAgent. Quitting the app stops serving (matches #0074).

## Where to look

- **New: `Issues/Remote/RemoteServer.swift`** — `NWListener` + a tiny route table. Gated `#if os(macOS)`.
- **No external dependency.** Use `Network.framework` (`NWListener`, `NWConnection`) directly. Don't pull in Vapor, swift-nio, or a third-party HTTP server — the route surface is small enough that a hand-rolled HTTP/1.1 parser over `NWConnection` is shorter and has no version-skew risk.
- **`Issues/State/IssueStore.swift`** — the server reads from the same store(s) the local app uses; no parallel data layer.

## Behavior

- **Bind**: by default listens on all non-loopback IPv4/IPv6 interfaces (per #0076 decision). A future issue can add interface pinning if needed.
- **Port**: configurable, default `0` (let the OS pick) on first run. Persist the chosen port to `UserDefaults` under `RemoteServer.port` so reconnects are stable across launches.
- **TLS**: not used in v1. The shared-secret auth + Tailscale's tunnel encryption (when Tailscale is the path) covers the threat model.
- **Concurrency**: each accepted `NWConnection` reads a single HTTP/1.1 request, dispatches, writes the response, and closes (or upgrades to WebSocket per #0100). Request handling is `Task { … }` per connection.
- **Auth middleware**: every request must carry `Authorization: Bearer iat_<…>`. Missing or unparseable header → `401 Unauthorized` with body `{"error":"missing_token"}`. Invalid token → `401 invalid_token`. Expired token → `401 expired_token`. Valid → request proceeds and `AccessToken.touch(hash:from:)` fires asynchronously so the read path isn't blocked on the keychain write.
- **Route table**: a simple `[Route]` matched by `(method, pathPattern)`. Patterns support `{folderId}` and `{id}` segment placeholders captured into a `[String: String]` for the handler. Unmatched routes → `404 not_found`.
- **Errors**: any thrown error from a handler maps to `500 internal_error` with the error description in the body in **debug builds only**; release builds return `500 internal_error` with no body.
- **CORS / preflight**: not handled. The viewer is a native client, not a browser.

## Public surface

```swift
@MainActor
final class RemoteServer {
    init(store: MultiFolderStore)        // see #0085 — the host serves multiple folders
    func start() throws                  // binds NWListener, begins accepting
    func stop()                          // tears everything down, closes connections
    var listeningPort: UInt16? { get }   // for the settings UI
    var connectedPeers: [PeerInfo] { get }  // for the connected-viewers list (#0092)
}
```

The server is owned by the same scene that owns the user's `IssueStore`s. It does not own its own copy of the data.

## Verification

- Toggle hosting on. `lsof -nP -iTCP -sTCP:LISTEN | grep Issues` shows the listener on the chosen port.
- `curl http://<ip>:<port>/v1/host` → `401 missing_token`.
- `curl -H "Authorization: Bearer iat_garbage" http://<ip>:<port>/v1/host` → `401 invalid_token`.
- `curl -H "Authorization: Bearer iat_<valid>" http://<ip>:<port>/v1/host` → `200` (handler returns once #0080 lands; for this ticket alone, a placeholder handler returning `{ "ok": true }` is fine).
- Toggle hosting off → port no longer listening.
- Quit the app while a connection is in flight → connection closes cleanly, no crash.

## Notes

- Hand-rolling HTTP/1.1 is fine for `GET` requests with small bodies. Keep the parser strict and return `400 bad_request` on anything weird (oversize headers, missing `Host`, body on a `GET`).
- Use `os.Logger` on `co.sstools.Issues` subsystem with category `RemoteServer` so `log stream` works the way it does for the rest of the app.
- The auth middleware is the load-bearing security boundary. It's worth a tight unit test pass before any handler logic exists.
