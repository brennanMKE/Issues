## Description

Add a **File → Print…** menu item (Cmd+P) that prints the currently-selected issue. macOS's standard print panel includes "Save as PDF" in its lower-left dropdown, so this single entry point covers both physical printing and PDF export without dedicated UI.

The menu item is **enabled only when an issue is currently selected**. With no selection, the item appears disabled in the File menu.

## Behavior

- Triggering Print presents the system print panel.
- The panel previews the selected issue rendered in a print-optimized SwiftUI view: light color scheme regardless of the user's app appearance, larger body font for readability on paper, no chrome (no toolbar, no tab bar, no detail panel), full-bleed margins handled by the print layout.
- Print panel's "Save as PDF…" option produces a clean PDF file the user can attach to a ticket / share with stakeholders.
- "Save as PDF…" defaults the filename to `NNNN.pdf` matching the issue's id.

## Implementation sketch

1. **Print-only view**: a new `IssuePrintView` that takes the issue and renders the same metadata + description shape as `IssueMarkdownSheet`, but with:
   - Forced `.environment(\.colorScheme, .light)`.
   - Larger body fonts (~13pt body, ~24pt H1) sized for printed pages.
   - No close button, no Quick Look hooks.
   - Page-aware layout: use `.padding(.horizontal, 36)` etc. for sane margins.

2. **Print pipeline**: SwiftUI on macOS exposes `.printAction` via the `.print(...)` modifier as of recent OS releases, but the most reliable cross-version path is to wrap the print call through AppKit:

   ```swift
   func printIssue(_ issue: Issue) {
       let view = IssuePrintView(issue: issue)
       let renderer = ImageRenderer(content: view)
       // …or use NSHostingView wrapped in NSPrintOperation:
       let hosting = NSHostingView(rootView: view)
       hosting.frame = NSRect(x: 0, y: 0, width: 612, height: 792) // US Letter pt
       let printInfo = NSPrintInfo.shared
       let op = NSPrintOperation(view: hosting, printInfo: printInfo)
       op.jobTitle = "\(issue.id).md"
       op.run()
   }
   ```

   The exact shape can be settled during implementation; the goal is "the print panel opens with our view and the suggested filename is `NNNN.pdf`."

3. **Menu wiring**: in `IssuesApp.commands`, add a `CommandGroup(replacing: .printItem) { ... }` (or after `.newItem`) that hosts a `Button("Print…", action: ...).keyboardShortcut("p", modifiers: .command).disabled(activeStore?.selectedIssue == nil)`. Route through `AppCommandsController.shared.printSelectedIssue?()`, set by `MainView.registerCommandHandlers()` to call `printIssue(_:)` with the active selection.

## Edge cases

- No selected issue → menu item disabled. Don't try to print "the most recent issue" or "the first visible issue" — be strict.
- Long descriptions: the print view should paginate cleanly. SwiftUI's `Text` inside a hosting view paginates correctly when the host's frame exceeds the page; the underlying NSPrintOperation handles the rest.
- Embedded screenshots (#0056): they should print at a reasonable size, not full original resolution. Cap inline width at ~480pt as in #0056.
- Markdown that references missing attachments: print view should show the same placeholder logic the on-screen view uses; don't crash mid-render.

## Verification

- File menu shows "Print…" with Cmd+P.
- With no row selected, "Print…" is greyed out.
- Select an issue, hit Cmd+P → print panel appears with a preview rendered in light mode.
- Save as PDF from the panel → file written with default name `NNNN.pdf`.
- The PDF opens cleanly in Preview / Quick Look and visually matches the on-screen detail panel content.
- Print on paper at standard quality (no need to actually print; "Save as PDF" exercises the same code path).

## Out of scope

- Printing a list of issues (table-style report). Single-issue print is the v1.
- Custom page sizes / orientation pickers in our UI — fall through to the system print panel's existing options.
- Including a header/footer with project name + page numbers — could be a follow-up.

## Root cause

Greenfield. No prior print path. The detail panel + markdown sheet were screen-only; PDF export went through the OS file-export menu of whatever app opened the `.md` file.

## Fix

Three small new pieces and a menu wire-up:

- **`Issues/Views/IssuePrintView.swift`** — print-only SwiftUI layout. Title at 24pt, metadata `Grid` rendered only for fields that have content, description at 13pt with 3pt line spacing. Forced `.environment(\.colorScheme, .light)` so the print rendering is unaffected by the user's app appearance. Padding 36pt all around.
- **`Issues/Services/IssuePrintRunner.swift`** — bridges into AppKit: wraps the SwiftUI view in `NSHostingView`, builds an `NSPrintInfo` with sensible defaults (36pt margins, automatic vertical pagination, horizontal fit), and runs an `NSPrintOperation` with `jobTitle = "<id>.pdf"` so the Save as PDF default filename matches the issue id. `.run()` synchronously presents the panel.
- **`AppCommandsController.printSelectedIssue`** — closure registered by `MainView` and called from the new menu item. Reading `activeStore?.selectedIssue == nil` in the menu's `.disabled(...)` keeps the item greyed out unless a row is selected.
- **`IssuesApp` File menu** — adds `Button("Print…") { commands.printSelectedIssue?() }` with `Cmd+P` after the existing command-palette entry.

## Files changed

## Gotchas

- **`NSPrintInfo.shared.copy()` (`as! NSPrintInfo`)** is the right way to fork the user's persisted print settings without mutating the global instance. Mutating `.shared` directly leaks across the app's lifetime.
- **`jobTitle` doubles as the Save as PDF default filename** — that's why we set it to `<id>.pdf` rather than the title. The user can rename in the panel; the default is what matters.
- **`@MainActor` on the runner.** `NSPrintOperation.run()` blocks the main thread waiting for the modal panel; calling from a background queue would deadlock SwiftUI's main-actor expectations.
- **Visual verification still needed.** The print panel is interactive — can't unit-test it. Real Save as PDF round-trip should confirm the filename defaults to `<id>.pdf`, the preview shows the issue laid out in light mode, and long descriptions paginate across pages.
- **The menu item lives in File, after the command palette entry**, matching macOS's standard placement.
