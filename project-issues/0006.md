## Description

The markdown sheet (`IssueMarkdownSheet`) currently uses a plain `Button("Close")` text button in its header. Replace the text label with an SF Symbols X icon so the close affordance is iconographic and matches the conventions of macOS modal sheets.

## Expected behavior

- The close control in the sheet header is an icon button using SF Symbols `xmark` (or `xmark.circle.fill` for a softer look — pick the one that fits the dark theme).
- The button keeps its existing keyboard shortcut binding to `.cancelAction` so `Esc` still dismisses.
- Provide an `.accessibilityLabel("Close")` and a `.help("Close")` tooltip since the visible label is gone.
- Sized to match the header bar height — roughly 18–22pt symbol, with subtle hover/pressed feedback.
- Tinted using the existing palette (`Color.appMuted` at rest, `Color.appText` on hover is one option).

## Notes

- Touch only `Issues/Views/IssueMarkdownSheet.swift` — the relevant block is the `Button("Close") { dismiss() }` around line 34.
- This change is just the markdown sheet's close button. The DetailPanelView's `✕` close (which uses a custom Text glyph) is out of scope; if the user later wants those unified, file a separate issue.
- Keep the rest of the sheet header layout intact: title + filename on the leading edge, close button trailing.
