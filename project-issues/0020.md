## Description

The app is dark-mode-only by design (`.preferredColorScheme(.dark)` on the root). PRD lists light mode as out of scope but it's a frequent ask for any indie macOS app — many users run light system-wide and a forced-dark window stands out.

## Notes

- Drop the `.preferredColorScheme(.dark)` and let the window honor `colorScheme`.
- Re-derive the palette in `Issues/Theme/Theme.swift` from `Color(NSColor.controlBackgroundColor)` etc. so light/dark adapt automatically. Audit `Color.app*` constants — every hex literal becomes a `Color(light:dark:)` pair.
- Status badge and timeline-bar opacities (`.background15`, `.background22`) read fine in dark; verify against light backgrounds and bump opacity if needed.
- Add a Settings entry "Appearance: Auto / Light / Dark" once the palette is dynamic.
