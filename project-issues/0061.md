# 0061 — French (fr) localization

| | |
|---|---|
| **Status** | open |
| **Module** | App |
| **Platform** | macOS |
| **First seen** | 2026-05-05 |

## Description

Translate every key in `Localizable.xcstrings` to French. Depends on #0059.

## Translation guidance

- Use the metropolitan-French / general convention (Apple ships a single `fr` locale; ne pas distinguer du Québécois).
- macOS app chrome is conventionally impersonal infinitive: "Ouvrir le dossier…", not "Ouvre le dossier…".
- Match Apple's French localization conventions: "Fermer" for "Close", "Recharger" for "Reload", "Réglages" for "Settings" (Apple's recent rebrand from "Préférences").
- Status display names: "Open" → "Ouverte", "In Progress" → "En cours", "Resolved" → "Résolue", "Closed" → "Fermée", "Won't Fix" → "Non corrigée".
- View modes: "Swimlanes" → "Couloirs", "Timeline" → "Chronologie", "List" → "Liste", "Recent" → "Récents".
- French uses spaces before some punctuation (`?`, `!`, `:`, `;`) — use the non-breaking space (`U+00A0`). Apple's localization team follows this; we should too. Most of our strings end in `…` (ellipsis, U+2026) which doesn't need the spacing rule.
- Pluralization: confirm `one` / `other` slots are filled (French only has those two for nouns, same as English).

## Process

Same as #0060: open the catalog, fill `fr` slots, run with `AppleLanguages = ("fr")`, walk every screen.

French strings tend to be ~20% longer than English. Watch the same fixed-width controls as in #0060.

## Verification

- Run with system language `fr` → every visible English string renders in French.
- Non-breaking spaces appear correctly before punctuation marks where French requires them.
- Pluralization renders correctly.
- Settings → Shortcuts pane labels fit without truncation.

## Out of scope

- Quebec French (`fr_CA`) variants — keep one `fr` slot for both.
- Localizing date formats — handled automatically by `DateFormatter` with the active locale.
