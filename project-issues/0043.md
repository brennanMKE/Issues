# 0043 — Drop List view's per-cell double-click open; rely on Enter / right-click

| | |
|---|---|
| **Status** | resolved |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |
| **Closed** | 2026-05-04 |

## Description

#0042 re-attached `.simultaneousGesture(TapGesture(count: 2)…)` to each `TableColumn` cell to restore the open-markdown affordance, omitting `.contentShape(Rectangle())` so the gesture wouldn't grab cell whitespace. That partially worked — double-click does open the sheet — but single-click row selection became unreliable again. Even with `.simultaneousGesture` (which fires alongside other recognizers rather than racing them), the count: 2 recognizer must still wait the macOS double-click interval before deciding the click was a single. During that wait, NSTableView's row-selection sometimes drops the click.

## Why this trade-off can't be split cleanly

SwiftUI's `Table` is backed by `NSTableView`, which exposes `doubleAction:` for reliable double-click handling. That property isn't surfaced by SwiftUI's `Table` API as of macOS 26.4. Every approach we've tried lives in user-space gesture recognizers that compete with — or delay — NSTableView's selection event:

| Approach | Single-click reliable | Double-click works |
|---|---|---|
| Original (`.onTapGesture(count: 2)` + `.contentShape(Rectangle())`) | ❌ | ✅ |
| `.simultaneousGesture` + `.contentShape(Rectangle())` | ❌ | ✅ |
| Strip all per-cell gestures; Table-level `.onTapGesture(count: 2)` | ✅ | ❌ |
| `.simultaneousGesture` only, no `.contentShape` | ❌ (degraded) | ✅ |

Single-click selection is the more important interaction — it drives both the detail panel and the keyboard navigation pivot — so we keep that reliable and drop the double-click affordance in List view specifically.

## Fix

Strip the per-cell `.simultaneousGesture(TapGesture(count: 2)…)` modifiers from every `TableColumn` cell. Cells return to plain `Text` + `.foregroundStyle` + `.frame(maxWidth: .infinity, alignment: .leading)`. Keep the Table-level `.contextMenu(forSelectionType: String.self)`.

Existing affordances still cover open-markdown in List view:
- **Enter** when a row is selected → opens the markdown sheet (wired in `MainView.onKeyPress(.return)` since #0011-ish).
- **Right-click → Preview Markdown** on the Table's context menu (auto-selects the row first).
- **Detail panel "NNNN.md ↗" link** at the bottom of the panel.

Other views (Recent, Swimlanes, Timeline) keep their double-click handlers because their rows are plain `HStack`s, not Table rows — no NSTableView underneath, no race.

## Verification

In List view:
- Single-click any row → reliably selects + detail panel slides in.
- Cmd+click selected row → deselects.
- Right-click any row → "Preview Markdown" works.
- Press Enter on a selected row → markdown sheet opens.
- Click "NNNN.md ↗" in detail panel → markdown sheet opens.
- Double-click on a row → no longer opens the sheet (intentional regression of #0042's behavior; documented above).

## Notes

If we ever want double-click in List view back, the canonical fix is wrapping NSTableView in `NSViewRepresentable` and using its `doubleAction:` selector. That's a much bigger change (re-implementing column sorting, row selection, and styling on top of AppKit) and out of scope for now.
