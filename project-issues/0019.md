## Description

A dedicated panel (or a footer banner) that lists files in the active folder which fail to parse or are inconsistent. Catches the failures the parser is currently strict-but-quiet about.

## Lints

- File matches `\d{4}\.md` but parser returns nil (missing title, missing Status row, hyphen instead of em-dash in title).
- Sibling folder `NNNN/` exists with no matching `NNNN.md`.
- `NNNN.md` references an attachment path that doesn't exist on disk.
- Status value not in the canonical set (`open`, `in-progress`, `resolved`, `closed`, `wontfix`).
- Duplicate ids (shouldn't happen given filenames, but worth checking content).

## Notes

- Run on every reload; cheap.
- Surface as a small "⚠ 3 issues" banner in `StatsBarView` that opens a sheet listing the offending files with one-line reasons.
- Don't auto-fix — that's the user's call. Provide "Reveal in Finder" per row.
