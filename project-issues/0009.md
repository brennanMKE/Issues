# 0009 — Per-tab persistence of filters, sort, view mode, and selection

## Description

Each tab should remember its filters (status / module / platform), view mode, sort column + direction, and selected issue id. Switching to a previously-opened tab restores its prior state instead of resetting to defaults; relaunching the app restores the same.

## Notes

- `IssueStore` already holds the fields. The gap is persistence across launches and survival when a tab is closed and reopened.
- Persist a small per-tab JSON blob into UserDefaults under `tabState`, keyed by the bookmark's resolved path (or a stable id derived from it). Restore in `TabsModel.restore()`.
- If a persisted `moduleFilter` no longer exists in the restored issues, fall back to nil — don't crash, don't filter to an empty result.
- Selection persistence is nice but lowest priority; filters and view mode matter more.
