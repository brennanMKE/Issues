## Description

Single-clicking an issue bubble (swimlane card) selects the issue and slides in the detail panel — that's the existing behavior and should stay. Add a **double-click** gesture on the same bubble that immediately presents the Textual markdown sheet (the one introduced in #0001), skipping the detail-panel intermediary.

## Expected behavior

- Single click on a swimlane `IssueCardView`: selects the issue (no change).
- Double click on a swimlane `IssueCardView`: presents `IssueMarkdownSheet` for that issue. Selection state can also update so the detail panel reflects which issue is open, but that's secondary.
- The two gestures must coexist cleanly — single-click selection should not feel laggy waiting for a possible second click. Use `.onTapGesture(count: 2)` alongside the existing single-tap; SwiftUI disambiguates these without an artificial delay on the single-tap path.
- Closing the sheet returns the user to the main view with their previous selection intact.

## Notes

- Entry point is `Issues/Views/IssueCardView.swift` — that's where the bubble's tap gesture lives.
- Sheet presentation is currently driven by `@State private var showingMarkdownSheet` in `Issues/Views/DetailPanelView.swift`. To open the sheet from a swimlane card, either:
  - Lift the sheet presentation up to a parent (e.g. `MainView` or `RootView`) keyed by an `Issue?` state, and have both `IssueCardView` and `DetailPanelView` set it; or
  - Have `IssueCardView` own its own `.sheet(isPresented:)` for the double-click path.
  Either is fine — the lifted version avoids two sheet presenters in the hierarchy if the user double-clicks while the detail panel is also showing the sheet, but it's slightly more wiring.
- Consider extending the same double-click behavior to:
  - `ListView` rows (probably trivial — `Table` supports `.onDoubleClick` patterns).
  - `TimelineView` bars.
  - The `RecentView` rows.
  All four selectable surfaces benefit equally. Out of scope to *require* all four for v1, but make the implementation reusable so adding the others later is small.

## Follow-up — extend to Timeline, List, and Recent

The first pass (commit `badcbf8`) wired the double-click only on `SwimlaneView`'s bubbles. The user expects the same gesture on the other three views as well. Thread the `onOpenMarkdown` closure down through `MainView` to:

- `TimelineView` — bars/rows that represent each issue.
- `ListView` — table rows (consider `.onTapGesture(count: 2)` on the row, or the `Table` selection-double-click idiom).
- `RecentView` — recent rows.

Selection behavior on a single click stays the same in all three. The double-click action calls `onOpenMarkdown(issue)` so the same `MainView`-owned `.sheet(item:)` presents.
- Make sure the double-click on a card doesn't double-trigger selection (single-click handler firing twice). If using the standard `.onTapGesture(count: 2)`, SwiftUI handles this; verify with the existing single-tap still being instant.
