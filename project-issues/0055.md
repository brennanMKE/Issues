## Description

Add a VS Code / Linear / Spotlight-style command palette that the user can trigger with **Cmd+Shift+P** (and via the **File → Show Command Palette…** menu item for discoverability). The palette is a small floating sheet with a text field and a results list. Type to fuzzy-match against:

- Every parsed issue (id + title), in the active tab.
- View-mode actions ("Switch to Swimlanes", "Switch to Timeline", …).
- Tab-switching actions ("Go to BucketKit", "Go to EurekaPlatforms", …).

Hit Enter to invoke the top result. Up/down arrows walk the results. Esc dismisses.

This is the single biggest discoverability win for keyboard-driven users — once they learn one shortcut, they get to every action without memorizing dedicated bindings.

## Implementation sketch

A new sheet view, `CommandPaletteView`, presented via a `@State var showingCommandPalette: Bool` on `MainView`. Triggered by:

- `CommandGroup(after: .newItem)` in `IssuesApp.commands` adds **Show Command Palette…** with the Cmd+Shift+P binding (also user-configurable via `ShortcutsStore` — extend `ShortcutAction` with a `.commandPalette` case).
- The menu item routes through `AppCommandsController.shared.showCommandPalette?()`, set by `MainView.registerCommandHandlers()`.

Inside the palette, the result list is a flat array of `Command` values:

```swift
enum Command: Identifiable {
    case issue(Issue)
    case viewMode(IssueStore.ViewMode)
    case tab(IssueStore)

    var id: String { /* derived */ }
    var displayText: String { /* "#0042 — Reply button…" / "Switch to Timeline" / "Go to BucketKit" */ }
}
```

Fuzzy matching: a small scoring function (Sublime/VS Code style — character matches, consecutive bonuses, prefix bonuses) is fine for v1. There's no need for a dependency.

## File menu integration

In `IssuesApp.commands`, add a `Button("Show Command Palette…")` to the existing `CommandGroup(after: .newItem)` block, with the configurable shortcut wired through `ShortcutsStore`. This makes the feature discoverable to users who don't yet know the keyboard shortcut.

## Edge cases

- Palette open with no results → show "No matches" muted text, not a blank list.
- Switching the active tab from the palette must not also dismiss the palette unexpectedly mid-typing — the palette closes only on Enter (invoke), Esc (cancel), or click-outside.
- Empty input shows the most recent issues (sorted by `modifiedAt` desc) and the standard view-mode/tab actions, so the palette is useful even before the user types.
- `selectedIssueID` updates through this path should bring the chosen issue into the detail panel and (optionally) open the markdown sheet — TBD which feels right; v1 picks "navigate to it, don't auto-open the sheet" so the palette feels precise.

## Verification

- Cmd+Shift+P → palette appears, focused.
- Type "reply" → results include any issue with "reply" in title or description; "Switch to … " actions filter out (no match).
- Enter on a result → palette dismisses, selection moves to that issue.
- File menu shows "Show Command Palette…" with the same shortcut suffix.
- Esc dismisses cleanly.
- Cmd+Shift+P from inside the markdown preview sheet also works (this needs the keyPress to fire from the sheet's focusable container too).

## Root cause

Net-new feature — there was no command palette in the app. Keyboard-driven users had to memorize one shortcut per action and there was no fast way to jump to an arbitrary issue by id or title fragment.

## Fix

Added a Spotlight-style sheet (`CommandPaletteView`) bound to **Cmd+Shift+P** and the new **File → Show Command Palette…** menu item. The palette fuzzy-matches a flat list of:

- every parsed issue in the active tab (id + title + module + description),
- view-mode actions (Switch to Swimlanes / Timeline / List / Recent — only the inactive modes are listed),
- tab-switching actions (Go to <repoName> — for every tab other than the active one).

Plumbing follows the existing `AppCommandsController` pattern: `IssuesApp.commands` adds a menu item that calls `triggerShowCommandPalette()`, and `MainView.registerCommandHandlers()` registers a closure that flips its `@State var showingCommandPalette` so the sheet only exists for the active scene.

A small `FuzzyMatch.score(query:candidate:)` scorer (Sublime/VS Code-style: prefix bonus, word-boundary bonus, consecutive-match bonus, gap penalty) ranks results when the field is non-empty. Empty input falls back to default ordering — recent issues first (sorted by `modifiedAt`), then view-mode actions, then tab actions — so the palette is useful before any keystroke.

Keyboard model:
- ↑/↓ walk the list (wrap-around) and `ScrollViewReader` keeps the highlighted row visible.
- Enter invokes the highlighted command.
- Esc dismisses without invoking.
- Empty result set shows a muted "No matches" line instead of a blank list.

Invoking an issue command sets `store.selectedIssueID` (no auto-opening of the markdown sheet — v1 picks "navigate, don't open"). View-mode commands set `store.viewMode`. Tab commands call `tabs.setActive(id:)`. The sheet dismisses on every invoke.

Also added a `.commandPalette` case to `ShortcutAction` (default `Cmd+Shift+P`) so the binding is user-configurable through the existing Settings → Shortcuts pane without any further wiring there — the pane iterates `ShortcutAction.allCases`.

## Gotchas

- **Cmd+Shift+P inside the markdown preview sheet was not solved.** macOS routes `.keyboardShortcut` from `.commands { … }` against the focused window/key view, and a presented sheet captures focus; firing menu shortcuts from inside another sheet would need an `.onKeyPress` proxy on `IssueMarkdownSheet`. Out of scope for this issue — leaving for a follow-up. Cmd+Shift+P from the main window works.
- `PaletteCommand` is `Identifiable` but **not** `Hashable`. The `.tab` case associates an `IssueStore` (a class without `Hashable`), so the `id` strings (`"issue:0042"`, `"view:timeline"`, `"tab:<uuid>"`) drive `ForEach` diffing instead.
- Default-ordered list shows the inactive view modes only. Listing the active mode would just be a no-op invoke; filtering it keeps the surface tight.
- `ShortcutsStore.reservedShortcuts` does not include Cmd+Shift+P, so the user is free to remap it through Settings without hitting the reservation guard.
