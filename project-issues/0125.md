## Description

With the design pivot in `#0124` (Mac mini as canonical host, Bonjour discovery dropped), the `RemoteFolderPickerView` no longer needs its "Discovered hosts" section. The viewer flow becomes:

1. Open the picker.
2. Type the host's address (Tailscale MagicDNS name like `mac-mini`, a `.local` mDNS name, or an IP) and port.
3. Paste a combined token (from `#0113`, includes the cert fingerprint).
4. Pick folders.
5. Connect.

This ticket removes the Bonjour-related UI scaffolding and tightens the manual-entry flow so it reads as the primary path, not a fallback.

## Steps to reproduce (current state)

1. Open Issues.app, **File → Connect to Remote Host…**.
2. Picker shows three sections: **Discovered hosts**, **Recent**, **Or enter manually**.
3. "Discovered hosts" is always empty in practice — Bonjour publish hasn't shipped (gated on `#0093`'s multicast approval). The section reads as broken.
4. The "Or enter manually" section is where the user always lands. Calling it a fallback is misleading; it's the only path.

## Expected behavior

1. Picker shows two sections: **Connect to a host** (the primary entry with host/port fields) and **Recent** (one-click reconnect to previously-used hosts).
2. A short helper line under the Host field: *Tip: if you're using Tailscale, the MagicDNS name (e.g. `mac-mini`) works from anywhere.*
3. No empty "Discovered hosts" section.

## Where to look

- **`Issues/Views/RemoteFolderPickerView.swift`** — the picker. Currently has a `DiscoveredHostsSectionView` (or equivalent) bound to `RemoteServiceBrowser.hosts`. Remove that section and its observers; promote the manual-entry section to the top.
- **`Issues/Remote/RemoteServiceBrowser.swift`** — delete the file. `NWBrowser` usage goes with it.
- **`Issues/Remote/RemoteProtocol.swift`** — `DiscoveredHost` struct can be deleted along with the browser, if no other code references it. `grep -rn "DiscoveredHost" Issues/` to confirm before removing.
- **`Issues/IssuesApp.swift`** — if the `Window("Connect to Remote Host", id: "remotePicker")` scene initializes a `RemoteServiceBrowser`, drop that wiring. The picker view no longer needs the browser as a dependency.
- **`Issues/Info.plist`** (or wherever it lives post-`#0110`) — `NSBonjourServices` can be removed. ATS-relevant keys (`NSAllowsArbitraryLoads` from `#0110`) stay until `#0114` removes them.

- The **Recent** list is the same persistence layer that exists today (`RememberedHost` keyed by hash of `host:port`, persisted via `RemoteSubscriptionStore` or its equivalent). Remove the `×` rows with a context-menu "Forget this host" entry — single-click forget per recent item.
- "Continue" runs the same connectivity probe as today (`GET /v1/host` over HTTPS — once `#0114` lands and the URL scheme flips to `https://`).

## Behavior

- **Discovery surface is zero.** No `NWBrowser`, no Bonjour, no multicast traffic. The app never publishes a service either (the host side's Bonjour publish from `#0088` is closed as `wontfix` per `#0124`).
- **Recent is the substitute for "Discovered hosts."** A previously-connected host appears for one-click reconnect; new hosts arrive via the manual-entry field.
- **No automatic discovery on `Window` appear.** The picker is fully manual; the user types or clicks Recent.
- **Validation flow unchanged.** Continue → `GET /v1/host` → 401 means "reachable, awaiting token" → advance to paste step (`#0096`); connection error → inline message; success → folder picker (`#0097`).

## Acceptance criteria

- "Discovered hosts" section removed from `RemoteFolderPickerView`.
- `RemoteServiceBrowser.swift` deleted; no remaining references.
- The Tailscale hint line is visible under the Host field.
- Recent list still works (existing persistence; just promoted to a top-level section instead of nested).
- All existing tests pass; any test referencing `DiscoveredHost` or `RemoteServiceBrowser` is updated or removed.
- Build clean with `xcodebuild`.

## Verification

- Open the picker on a fresh install. No empty "Discovered hosts" section visible.
- Type `mac-mini:51823` (or whatever the Mac mini's address is), Continue → reaches the token paste step.
- Connect successfully → Recent populates on next open.
- Click a Recent row → reconnect succeeds (or surfaces the expected token-revoked banner if the host rotated tokens).
- Right-click / context-menu a Recent row → "Forget this host" removes the row.
- `grep -rn "Bonjour\|NWBrowser\|_issues._tcp\|DiscoveredHost" Issues/` returns no hits in non-issue-tracking files.

## Notes

- The Recent list is more useful than Bonjour ever was for this app's single-user, few-hosts pattern. Bonjour browsing was designed for "I might have N hosts on my network, show them all"; the user has at most three, and Recent already captures them after first use.
- The Tailscale hint is in copy, not code — no Tailscale dependency or detection logic. The user knows whether they're on Tailscale; we just remind them the MagicDNS name works.
- The picker's primary path is **type-and-paste**. Reasonable; everything else in the app's flow is also "user knows what they want, app gets out of the way."
- Deleting `RemoteServiceBrowser` removes the `com.apple.developer.networking.multicast` *need* on the viewer side. The entitlement was never granted (`#0093` blocked), so removing the dependency removes the blocker. The viewer's entitlements file shrinks accordingly.

## Scope (out)

- A "search known networks" or "scan Tailnet for hosts" feature. Out of v1; Recent covers the actual need.
- Auto-fill of the host field based on previous connections. Recent does this implicitly with one click.
- Removing the `#0110` ATS exception. That's `#0114`'s job once cert pinning lands.
- Picker layout changes beyond removing the Bonjour section and adding the Tailscale hint. Bigger redesigns are out of scope.

## Dependencies

- **Gated on `#0124` decision** being accepted.
- **Not blocked on `#0111`–`#0115`** (the TLS work). The picker UI cleanup can land before TLS; the picker just keeps using `http://` (via `#0110`'s ATS exception) until `#0114` flips it.
- **Doesn't block iPhone work (`#0108`).** The iPhone picker, when it lands, reuses the same view file and inherits these cleanups for free.

## Root cause

Not a bug — design pivot. The Bonjour discovery cluster (#0088/#0089/#0090/#0093) was always blocked on Apple's multicast entitlement, and the structural decision in #0124 made the entire discovery surface unnecessary: one canonical host (the Mac mini) reached via Tailscale MagicDNS, `.local`, or a typed-in IP. The empty "Discovered hosts" placeholder section in the picker had been visible since the original UI shipped and reads as broken regardless of the user's network.

## Fix

- Removed the `section(title: "Discovered hosts")` block from `RemotePickerHostPhase`. The "No hosts discovered" empty-state copy went with it.
- Renamed the third section's label from "Or enter manually" to "Connect to a host" and moved it to the top — direct addressing is the primary path now, not a fallback.
- Added a small caption line under the host/port fields: *"Tip: if you're using Tailscale, the MagicDNS name (e.g. `mac-mini`) works from any network."* No Tailscale dependency or detection logic — copy only.
- Deleted the `discovered: [RemoteHostIdentity]` property from `RemoteFolderPickerModel`. `grep -rn 'discovered\|RemoteServiceBrowser'` in `Issues/` and `IssuesTests/` shows no other references, so no follow-up cleanup was needed.
- Updated the file-header doc comment to reflect that "Discovered hosts" is gone and to point at #0125/#0124 for context.
- Edited `Info.plist`'s comment blocks: the placeholder note about `NSBonjourServices` landing once the multicast entitlement comes through is replaced with a wontfix reference to #0124. The Info.plist itself doesn't change shape (no Bonjour keys were ever present).

`RemoteServiceBrowser.swift` was not deleted — it never existed. The picker carried a stub `discovered` array that was always empty; the browser was waiting on the multicast entitlement that never arrived. The acceptance criterion was effectively met as soon as the picker section was removed.

## Files changed

- `Issues/Views/RemoteFolderPickerView.swift` — removed the "Discovered hosts" section + the `discovered` property; renamed the manual-entry section to "Connect to a host" and moved it to the top; added the Tailscale tip line; updated the doc comment.
- `Info.plist` — reworked the Bonjour-related comments to reflect the wontfix; no key changes.

## Gotchas

- The Tailscale hint sits inside the section header block rather than as a fully separate row so it lines up with the host/port fields' indentation. If a future redesign breaks that visual relationship, consider promoting it to its own row with a leading `info.circle` icon.
- The Recent list is the only surviving "found host" affordance. If the user never connected before, they have nothing in Recent and the entire picker is "type address, paste token, pick folders" — which is the intended v1 flow. No additional empty-state CTA was added.
- The `RemoteHostIdentity` struct is still used by `RemoteHostRecents`; deleting `discovered` didn't ripple into that type. If the codebase ever picks up a different "found host" affordance (e.g. a Tailscale-side enumeration), reuse the existing type rather than reintroducing a parallel one.
