# 0045 — Spacebar dismisses the markdown preview sheet (Quick Look toggle)

| | |
|---|---|
| **Status** | in-progress |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |

## Description

#0044 added spacebar to *open* the markdown preview from any view mode, matching the Finder/Mail/Music Quick Look gesture. The full Finder convention also has space *close* the preview when one is showing — press space, peek; press space again, return to the row list.

The open path works because `MainView.contentArea` is `.focusable()` and owns key events while the main UI has focus. Once the sheet is presented, focus moves into the sheet and contentArea's `.onKeyPress` no longer fires — so without a complementary handler on the sheet itself, space inside the sheet does nothing.

## Fix

Add a `.focusable()` + `.onKeyPress(.space)` pair on `IssueMarkdownSheet`'s root `VStack` that calls `dismiss()`:

```swift
// IssueMarkdownSheet.swift, on the outer VStack
.focusable()
.focusEffectDisabled()
.onKeyPress(.space) {
    dismiss()
    return .handled
}
```

`.focusEffectDisabled()` keeps the focus ring off the sheet's outer container — same pattern `MainView.contentArea` uses. The existing close button (`.keyboardShortcut(.cancelAction)`, i.e. Esc) stays as the alternative dismiss path.

## Behavior matrix after the fix

- Sheet closed, row selected, press space → sheet opens (#0044).
- Sheet open, press space → sheet dismisses (this issue).
- Sheet open, press Esc → sheet dismisses (unchanged).
- Sheet open, click into the rendered markdown to select text, then press space → the focus is on the focusable VStack, not the text selection target, so space still dismisses. This matches Finder Quick Look: any spacebar press while the preview is showing closes it, regardless of where the cursor is in the preview content.

## Verification

Open the app, pick any view mode:
1. Select a row (single-click or ↑/↓).
2. Press space → preview opens.
3. Press space again → preview closes.
4. Repeat in fast succession → opens and closes reliably.
5. Open preview, scroll the markdown content with the trackpad → scrolling still works (sheet's own ScrollView owns the gesture).
6. Open preview, press Esc → still closes.
