# 0041 — Detail panel body doesn't update when selected issue changes

| | |
|---|---|
| **Status** | in-progress |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |

## Description

When the user picks a different issue in any view (List, Recent, Swimlanes, Timeline) the right-side detail panel updates its header (id + title) and metadata (status, module, platform, filed/closed) but the body section at the bottom keeps showing the previously-selected issue's markdown content. Switching back-and-forth between two rows leaves the body permanently stuck on whichever was selected first.

## Root cause

`Issues/Views/DetailPanelView.swift` renders the body via Textual's `StructuredText(markdown:baseURL:)` view inside its `description` computed property. The header / metadata sections re-evaluate cleanly because they're driven by simple `Text(issue.title)` etc. — SwiftUI updates them whenever `issue` changes.

`StructuredText` appears to retain its parsed/laid-out attributed content as internal state and does not invalidate when its `markdown:` initializer parameter changes between renders of the same view identity. Same view at the same position + new prop value ≠ rebuild from scratch — it's just a prop update, and Textual's view ignores the new prop for its already-rendered content.

## Fix

Force SwiftUI to treat the description subtree as a fresh view whenever the issue changes, by attaching `.id(issue.id)` to the `description` body. SwiftUI sees the id change, discards the old subtree (including Textual's internal state) and instantiates a new one against the current markdown.

```swift
// DetailPanelView.swift, in `description`
private var description: some View {
    Group {
        if let body = bodyMarkdown() { ... }
        else if !issue.description.isEmpty { ... }
        else { ... }
    }
    .id(issue.id)
}
```

`issue.id` is the 4-digit string from the filename — stable, unique per issue, and changes exactly when the user picks a different row.

## Verification

After the fix:
- Click row A → detail panel shows A's body.
- Click row B → detail panel shows B's body in full (not just the header).
- Click back to A → A's body is back.
