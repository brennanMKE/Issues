# 0013 — macOS notification when issue files change while the app is in the background

## Description

When the app is not in the foreground and a watched folder gets a new issue, a removed issue, or a status change, post a macOS user notification. Skip notifications while `NSApplication.shared.isActive == true` — the unseen-changes dot from #0003 is enough when the user is looking. Tapping the notification activates the app and switches focus to the tab whose folder changed.

## Expected notification format

- **Title**: `#NNNN — <issue title>`, e.g. `#0042 — Reply button not functional on post cells`.
- **Subtitle / body**: short description of the change — `New issue in BucketKit`, `Status: open → in-progress`, or `Removed from BucketKit`. Include the repo (tab) name so the user knows which folder it came from at a glance.

## Notes

- Use `UNUserNotificationCenter`. Request authorization on first run; if denied, the dot indicator continues to be the only signal.
- Hook into the same `IssueStore.onReload` callback that drives the unseen indicator (#0003). Compute the diff between the previous and new snapshots; emit one notification per changed issue (with sane coalescing — see below).
- The user identifier on each `UNNotificationRequest` should embed the tab id and issue id so the click handler knows which tab to focus and which issue to select. On notification tap, call `tabs.setActive(id:)` and `store.selectedIssueID = …`.
- Coalesce bursts: debounce 2s after a reload before deciding what to notify about, so a save that touches multiple files emits at most one notification per file. If more than ~5 files change at once (e.g. a git pull), collapse into a single "BucketKit · 8 changes" notification instead of spamming.
- App must be configured as a `UNUserNotificationCenterDelegate`; the delegate's `didReceive` handler is where the tab-focus action runs.
- Verify behavior under the existing sandbox entitlements; `UNUserNotificationCenter` should work on macOS without additional entitlements but this is worth checking on first run.
