# 0033 — `Task.sleep(nanoseconds:)` should be `Task.sleep(for:)`

| | |
|---|---|
| **Status** | in-progress |
| **Module** | State |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |

## Description

`Issues/State/TabsModel.swift:437` uses the legacy nanosecond-based sleep API:

```swift
try? await Task.sleep(nanoseconds: UInt64(Self.persistDebounce * 1_000_000_000))
```

Modern Swift Concurrency replaces this with the duration-based variant, which is type-safe (no manual nanosecond math) and reads more clearly.

## Suggested change

```swift
try? await Task.sleep(for: .seconds(Self.persistDebounce))
```

`persistDebounce` is already a `TimeInterval` (a `Double` of seconds), so `.seconds(_:)` accepts it directly.

## Notes

- `swiftui-pro/references/swift.md`: "Never use `Task.sleep(nanoseconds:)`; use `Task.sleep(for:)` instead."
- Behavior is identical — this is a clarity-only change.
