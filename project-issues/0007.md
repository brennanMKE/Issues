# 0007 — Full-text search across issues with Cmd+F

| | |
|---|---|
| **Status** | open |
| **Module** | Views / State |
| **Platform** | macOS |
| **First seen** | 2026-05-03 |

## Description

Add a search field that filters the active tab's issue list by matching the query against the title and the Description's first paragraph. Cmd+F focuses the field; Esc clears. Search AND-combines with the existing status / module / platform filters.

## Notes

- v1: case-insensitive substring match on `title` + `description`. A "search body" toggle (matches the full file contents) can come later.
- `IssueStore` is the right home for the query state and `filteredIssues` derivation — extend the existing filter logic.
- Search UI lives in `ToolbarView`, leading edge, before the status pills.
- Cmd+F requires a `CommandGroup` on the menu so the shortcut routes correctly. The app currently has no custom menu — small refactor.
- No match highlighting in v1; just filter the list.
