# 0034 — `HelpView.loadMarkdown` mutates `@State` during view body evaluation

| | |
|---|---|
| **Status** | resolved |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |
| **Closed** | 2026-05-04 |

## Description

`Issues/Views/HelpView.swift:102-127` reads bundled markdown synchronously inside view-body evaluation and writes the result into `@State var contentCache` from the same call:

```swift
private func loadMarkdown(for section: HelpSection) -> Result<String, Error> {
    if let cached = contentCache[section.id] { return .success(cached) }
    ...
    contentCache[section.id] = text     // <-- @State mutation during body evaluation
    return .success(text)
}
```

`sectionView(for:)` calls this from inside its `switch` (line 58). Mutating `@State` during body evaluation is fragile — SwiftUI may invalidate the view mid-render and is documented as undefined behavior. The current comment in the source already acknowledges this:

> SwiftUI tolerates state mutation from a view's body during a switch evaluation here because the cache write doesn't change which branch we render

That tolerance is incidental, not guaranteed.

## Suggested change

Drive the load from `.task(id: selection)` so each section change kicks off an async load that lands in `contentCache` from outside the body. Render `nil` while loading, success/failure once the task completes.

```swift
@State private var contentCache: [String: Result<String, Error>] = [:]

var body: some View {
    NavigationSplitView { sidebar } detail: { detail }
        .background(Color.appBackground)
        .task(id: selection) {
            await loadIfNeeded(selection)
        }
}

@ViewBuilder
private var detail: some View {
    if let result = contentCache[selection] {
        sectionView(for: result)
    } else {
        ProgressView()
    }
}

private func loadIfNeeded(_ id: HelpSection.ID) async {
    if contentCache[id] != nil { return }
    guard let section = HelpCatalog.sections.first(where: { $0.id == id }) else { return }
    contentCache[id] = await Task.detached(priority: .userInitiated) {
        // ... existing load logic, returning Result
    }.value
}
```

(Sketch — the exact surface can vary; the goal is to keep the body pure.)

## Notes

- The bundled markdown files are tiny so the cache miss is invisible in practice; this is a correctness fix rather than a performance one.
- Eight sections, each only loaded once — `Task.detached` is overkill; a plain `Task { @MainActor in ... }` reading from disk is fine on macOS.
- Worth verifying that switching sidebar selections still feels instant after the refactor.
