## Description

Phase 3 of the remote-access plan (#0076). Today every tab points at a local folder via `IssueStore` + `LocalFolderIssueSource`. Once Phase 3 lands, a tab can also point at a remote folder via `RemoteHostIssueSource` (#0094). The view stack handles both transparently because the source is abstracted (#0077), but the tab needs to know it's remote so it can persist correctly, show a small indicator, and route the right UI affordances.

## Where to look

- **`Issues/Models/TabPersistedState.swift`** ‚Äî extend the persisted shape to record local-or-remote, host id, folder id.
- **`Issues/State/IssueStore.swift`** ‚Äî already source-agnostic; takes either a `LocalFolderIssueSource` or `RemoteHostIssueSource`.
- **`Issues/Views/`** wherever tabs render ‚Äî add a small "remote" glyph next to the tab title.
- **`Issues/State/RemoteSubscriptionStore.swift`** (#0098) ‚Äî the source of truth for what remote tabs should be open at launch.

## Persisted shape

```swift
enum TabSource: Codable, Equatable {
    case local(bookmarkData: Data)               // existing
    case remote(hostId: String, folderId: String) // new
}

struct TabPersistedState: Codable {
    let id: UUID
    var source: TabSource
    var displayName: String         // last-known display; refreshed on connect
    // existing fields: filters, view mode, sort order, etc.
}
```

- Tokens are not persisted in the tab ‚Äî they live in the Keychain (#0095), looked up by `hostId` on reconnect.
- The bookmark vs. host-id split keeps local and remote tabs cleanly distinguishable in the persisted JSON.

## UI

- Tab title shows the folder's display name (from `project.json` `name` via #0075, or parent path).
- A small icon (e.g. SF Symbols `network`) appears to the left of the title for remote tabs. No icon for local tabs.
- On hover, a tooltip shows `Connected to <hostDisplayName>` or `Disconnected ‚Äî reconnecting‚Ä¶` while the source is in a disconnected state.
- A remote tab's overflow menu adds:
  - **Manage subscriptions‚Ä¶** (opens #0098)
  - **Forget this host** (proxies to the same removal in #0098)
  - **Open repository** (if `project.json` `repository` is set)

## Lifecycle

- **Open**: created by the picker (#0097) on Confirm. Construction:
  1. Build a `RemoteHostIssueSource` (#0094) with `(host, port, token, folderId)`.
  2. Build an `IssueStore` around it.
  3. Append a new `TabPersistedState(source: .remote(hostId, folderId), ‚Ä¶)`.
- **Persist**: the existing tab persistence path serializes `source` as either `.local` or `.remote`; no other code changes.
- **Reopen on launch**: the existing tab restore path constructs each tab's source based on `source`. For `.remote`, look up the token via `ViewerTokenStore` (#0095) and resolve the host's `(ip, port)` from either:
  - The Bonjour browser (#0089) once a matching service appears, or
  - The persisted manual entry from #0091.

  Reconnect logic itself is #0103. For this ticket, just unblock the construction path.
- **Close**: tearing down the tab cancels the source, drops the WS subscription, and removes the tab from `RemoteSubscriptionStore` if it's the last one for its host (the user can re-add via the manage sheet).

## Verification

- Pick two folders in #0097's flow ‚Üí two new tabs appear with the network icon. Each renders its folder via the same swimlane / list / timeline / recent code as a local tab.
- Quit and relaunch ‚Äî the same two remote tabs reappear. They show "reconnecting‚Ä¶" briefly, then connect and load.
- Right-click a remote tab ‚Üí overflow menu has Manage Subscriptions, Forget Host, Open Repository (when applicable).
- Close a remote tab ‚Äî the WS unsubscribe fires (#0102), tab disappears, persistence updates.

## Notes

- Don't store the full `(host, port, token)` triple in the tab ‚Äî the host can move (Bonjour name resolves to a different IP), the token can rotate, and the folder id is the stable handle. Token via Keychain, host via Bonjour or the persisted manual entry.
- The "remote indicator" glyph is intentionally subtle; loud indicators are noise once the user has remote tabs as a normal part of the workflow.

## Scope (out)

- Reconnect logic on launch ‚Äî #0103.
- Disconnect indicator UI when the connection drops mid-session ‚Äî #0104.
- Find-by-name fallback for changed folder ids ‚Äî handled in #0103.

## Resolution

Landed alongside the picker in the bundled viewer-UI commit `4635e4b` (merge `acb3876`). The remote-tab type isn't a separate enum ‚Äî `IssueStore` is source-agnostic post-#0077, so a remote tab is just an `IssueStore(source: RemoteHostIssueSource(...))`. The detection points are:

- **`TabsModel.openRemoteTab(host:port:token:folderId:displayName:)`** ‚Äî constructs the source + store + attaches the reload hook + persists. Dedupes against existing `(host, port, folderId)` triples so a re-open activates the existing tab.
- **`IssueStore.isRemote` / `remoteEndpoint`** ‚Äî convenience accessors that downcast the source for the UI layer; `folderId` falls back to `RemoteHostIssueSource.folderId` for sources without a security-scoped bookmark.
- **`RemoteHostIssueSource.urlScheme = "issues-remote"`** static ‚Äî `TabChipView` checks `store.folderURL.scheme == RemoteHostIssueSource.urlScheme` to render the `antenna.radiowaves.left.and.right` indicator.
- **Persistence** ‚Äî sibling `openRemoteTabs` UserDefaults key with a `RemoteTabPersistence` array of `(host, port, folderId, displayName)`. Restore re-looks-up the bearer from `ViewerTokenStore`; entries without a stored token are dropped with a log line. Local tab persistence under `openTabs` is unchanged.

See `project-issues/0097.md`'s resolution for the full picker writeup; #0099 lives inside that surface.
