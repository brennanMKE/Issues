## Description

The detail panel today shows only the parsed `## Description` paragraph as plain `Text`. Render the full markdown body — Steps to reproduce, Expected, Actual, Notes, Attachments — using the same Textual `StructuredText` view that powers the markdown sheet (#0001). The user gets the whole context without needing to open the sheet.

## Notes

- Read the file's raw contents (already on disk; the security scope is active for the active tab).
- Reuse `StructuredText(markdown: text, baseURL: issue.fileURL.deletingLastPathComponent())` so attachment images resolve. Strip the metadata table and title at the top before rendering — the panel already shows them in its own layout.
- Wrap in a `ScrollView`; the panel is 360pt wide, so font sizes need to come down (~12pt body, ~13pt section headings).
- The double-click → markdown sheet remains useful for a roomier read; this just makes the panel useful for the 80% case.
