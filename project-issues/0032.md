# 0032 — Replace inline `Binding(get:set:)` with `@Bindable` pass-through bindings

| | |
|---|---|
| **Status** | in-progress |
| **Module** | Views |
| **Platform** | macOS |
| **First seen** | 2026-05-04 |

## Description

Three sites build manual `Binding(get:set:)` wrappers in view bodies even though the `IssueStore` is already exposed via `@Bindable`. SwiftUI's `@Bindable` produces a `Binding` for any settable property via `$store.<property>`, so the manual wrappers are unnecessary indirection.

- `Issues/Views/ToolbarView.swift:107-119` — `moduleBinding`, `platformBinding`
- `Issues/Views/ListView.swift:102-107` — `selectionBinding`

`store.moduleFilter`, `store.platformFilter`, and `store.selectedIssueID` are all `var` (settable) on the `@Observable` `IssueStore`. They can be passed directly as `$store.moduleFilter`, `$store.platformFilter`, `$store.selectedIssueID`.

## Suggested change

```swift
// ToolbarView.swift — Before
Picker("Module", selection: moduleBinding) { ... }
Picker("Platform", selection: platformBinding) { ... }

// After (delete moduleBinding / platformBinding helpers entirely)
Picker("Module", selection: $store.moduleFilter) { ... }
Picker("Platform", selection: $store.platformFilter) { ... }
```

```swift
// ListView.swift — Before
Table(sortedIssues, selection: selectionBinding, sortOrder: $sortOrder) { ... }

// After (delete selectionBinding helper)
Table(sortedIssues, selection: $store.selectedIssueID, sortOrder: $sortOrder) { ... }
```

## Notes

- `swiftui-pro/references/data.md`: "Strongly prefer to avoid creating bindings using `Binding(get:set:)` in view body code."
- After the swap, the helper computed properties become dead code — delete them.
- `IssueStore` is `@Observable` `@MainActor` and these properties are already `var`, so `@Bindable` produces the binding for free.
