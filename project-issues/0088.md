# 0088 — Bonjour publish: NWListener service for _issues._tcp.local.

| | |
|---|---|
| **Status** | open |

| **Platform** | macOS |

## Description

Phase 2 of the remote-access plan (#0076). Once the host is serving (#0079), it should advertise itself on the local network so viewers don't have to type IPs. Bonjour is the obvious fit: zero-config, the OS handles broadcast, and `Network.framework` exposes it on a single API surface.

## Where to look

- **`Issues/Remote/RemoteServer.swift`** (from #0079) — already owns the `NWListener`. Add `NWListener.service` configuration and lifecycle.
- **`Issues/Info.plist`** — add `NSBonjourServices` (handled in #0093).
- **`Issues/Issues.entitlements`** — add `com.apple.developer.networking.multicast` (also #0093).

## Service configuration

- **Type**: `_issues._tcp` (full DNS form `_issues._tcp.local.`).
- **Name**: defaults to the host's display name from #0083 (e.g. `Brennan's MacBook Air`). When two hosts collide, mDNSResponder appends ` (2)` automatically; we don't need to handle the collision.
- **Domain**: `nil` — let the OS use the default (`local.`).
- **Port**: same port the HTTP listener bound (#0079).
- **TXT record**:
  ```
  v = 1                  ← protocol version (Int)
  n = "Brennan's MacBook Air"   ← display name
  c = 3                  ← folder count (Int)
  ```
  - Why these keys: short keeps the record small (TXT has a 255-byte cap per string). Folder names don't go in TXT — fetched on demand via `/v1/folders` after the viewer connects.
  - The `c` value lets the picker show a `… · 3 folders` subtitle without round-tripping.

## Behavior

- Service publishing **starts when the listener starts** (toggle on in #0083) and **stops when the listener stops**.
- TXT record updates when:
  - Display name changes (#0083) — re-publish.
  - Folder count changes (folder added/removed in tabs, or a per-folder toggle in #0085 flips) — re-publish.
- Listener and Bonjour share the same `NWListener` instance; setting `listener.service = NWListener.Service(name: …, type: "_issues._tcp")` is the one-liner that ties them together.
- If the multicast entitlement is missing, `NWListener.start` succeeds for HTTP but Bonjour publishing silently doesn't propagate. Log a warning at startup if multicast isn't entitled (read from the running entitlements via `SecCodeCopySigningInformation`).

## Verification

- `dns-sd -B _issues._tcp` from another Mac on the same Wi-Fi → host appears within ~2 seconds.
- `dns-sd -L "Brennan's MacBook Air" _issues._tcp` → resolves to `<hostname>.local` + port. TXT shows `v=1 n=… c=N`.
- Toggle hosting off → service disappears from the browse list within a second.
- Add a folder and toggle it on (#0085) → TXT record updates with the new `c`.
- Run on a Mac without the multicast entitlement → log shows the warning, but viewers can still connect via the manual `host:port` field (#0091).

## Notes

- We're publishing **one Bonjour record per host instance**, not per folder. Per `RemoteAccess.md`: folders are listed via `GET /v1/folders` after a viewer connects. This keeps the picker UI clean and avoids re-broadcasting on every folder add/remove.
- Tailscale traverses Bonjour only with MagicDNS enabled. That's a Tailscale-side setting, not something we configure. Without MagicDNS over Tailscale, viewers fall back to the manual `host:port` field (#0091). Verification of both paths is #0090.
