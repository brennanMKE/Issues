## Description

Umbrella tracking issue for the remote-access feature described in `RemoteAccess.md`. Goal is to view a folder of issues on Mac A from Issues.app running on Mac B, with no port forwarding and no shared Apple ID assumption — using GitHub-PAT-style access tokens for auth and HTTP/WebSocket over the local network (LAN, Tailscale, or any overlay that gives both ends a routable IP).

This issue exists so the cross-cutting concerns (entitlements, naming, scope, phasing) have a single home and so the per-phase issues below can link back to one place. Close this when the Mac side is shipped and stable; iPhone work is gated behind a separate tracking issue (#0108).

## Decisions locked 2026-05-08

- **Naming** — Host (the Mac with the folders) and Viewer (the device connecting). May rename later if it grates.
- **iCloud token sync** — out for v1. The viewer stores its token in its own Keychain. The user transfers tokens by paste (Notes will sync between Macs if needed).
- **Folder picker** — must support "1 / subset / all" with one-click select-all and per-row toggle.
- **Bind behavior** — listen on all non-loopback interfaces by default. The auth layer + the user's network (LAN or Tailscale) is what makes it safe; an interface-pinning toggle is deferred to v2.
- **Testing rig** — install signed/notarized DMG on a remote Mac; drive that Mac via Screen Sharing while running Issues.app locally. Both Macs reachable over Wi-Fi LAN and over Tailscale. iPhone testing happens after the Mac side is done.

## Decisions locked 2026-05-11

- **#0077 (IssueSource refactor) is approved to start.** User reviewed the approach and gave the green light.
- **Phase 0 + Phase 1 are go.** Phase 0 lands and bakes first; Phase 1 follows as a sibling PR (not bundled).
- **`project.json` may not be present** in a watched folder. Defaults match today's behaviour: name = parent folder name, no URL. Every ticket that reads `project.json` must treat the file as optional and fall back silently — no warning surfaced to the user for "file missing" (only for "file present but malformed", per #0075).
- **`description` field is a planned IssuesSkill extension**, not in the live schema yet. Today's `project.json` carries `name` + `url` (per #0075). The wire format keeps `description: String?` on `FolderInfo` so it lights up automatically once the skill adds it, but for v1 it'll always be `nil`.
- **`FolderInfo` field name is `url`, not `repository`** — aligns with the live `project.json` schema.
- **Attack order is locked:** #0077 → #0078 → #0079 → #0080 → #0081 → #0082 → #0083 → #0084 → #0085 → #0086 → #0087. Phase 2 starts as soon as #0093 is unblocked; the Phase 0+1 tickets above do not depend on it.
- **#0093 unblock action** — not a cert install. The Mac mini needs to be registered at developer.apple.com → Devices via its Provisioning UDID. See #0093 for the corrected wording.

## Notes

- Single user, single Apple ID assumption is load-bearing. If we ever want multi-user shared access, the auth model needs a redesign and this whole tree gets revisited.
- Read-only by design. No write semantics on the wire, no on-disk caching on the viewer.
- The server lives inside `Issues.app`; quitting the app stops serving (matches #0074).
