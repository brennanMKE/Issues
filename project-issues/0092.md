## Description

The host should show a live list of currently-connected viewers вҖ” useful for confirming a viewer authenticated, spotting stale connections, and (eventually) revoking access mid-session. - One row per active connection (HTTP keep-alive *or* WebSocket once #0100 lands; for v1 mostly WS).
- Token column shows the **token name** (#0084), not any of the secret material. Resolved via the token's hash on connect.
- Address column shows the IPv4/IPv6 (or reverse-DNS if cheap to fetch вҖ” best-effort, no blocking).
- Subscribed column lists folder display names from the WS subscribe message (#0100). Truncates with a tooltip on hover when the list is long.

## Behavior

- The list updates in near-real-time as connections open and close. No manual refresh.
- HTTP-only requests (no WS attached) appear briefly and disappear when the response writes. To avoid flicker we include only connections that have either upgraded to WS or have an outstanding HTTP request older than 200 ms.
- Right-click on a row вҶ’ "Revoke this token" (jumps to the access-token list with that token highlighted) and "Disconnect" (closes just this connection). Disconnect is a soft action вҖ” the viewer will reconnect immediately if its token is still valid; revoke is the permanent option.

## Verification

- Connect from another Mac via the smoke-test CLI (#0087) вҖ” a row appears.
- Stop the CLI вҖ” row disappears within a second.
- Open the Mac viewer (once #0099 lands) вҖ” row appears, Subscribed column shows the folder names.
- Right-click вҶ’ Revoke вҶ’ confirm in the token list. The viewer's WS closes within a second; row goes away.

## Notes

- Don't surface internal connection ids, request paths, or anything else that's not directly useful. The columns above are enough.
- Avoid auto-scrolling on update вҖ” if there are more viewers than fit, let the user scroll the list manually.

## Scope (out)

- A full audit log of past connections вҖ” out of v1; the per-token `lastUsedAt` / `lastUsedFrom` fields (#0078) cover the common need.
- "Kick + ban" (revoke + block this IP) вҖ” overkill for single-user.
- Per-folder fan-in counts ("3 viewers watching folder X") вҖ” interesting but not load-bearing.
