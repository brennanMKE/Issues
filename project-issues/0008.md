# 0008 — Keyboard shortcuts for selection, tabs, and view actions

## Description

Power-user keyboard shortcuts so the app is drivable without a mouse. Today only Esc (dismiss sheet) is wired.

## Expected behavior

- **Selection**: ↑/↓ moves through the visible issue list and updates `selectedIssueID`. Most useful in **List** and **Recent** views, especially when the detail panel is open on the right — the user can hold ↓ and watch each issue's detail render in turn. Enter opens the markdown sheet for the selected issue. Should also work in Swimlanes (cycle through the flat sorted issues) and Timeline (move through bars in display order).
- **Tabs**:
  - **Cmd+1…9** jumps to the Nth tab in left-to-right order. Tabs aren't visibly numbered, but Safari/Chrome have trained users to expect this; matching the convention is cheap.
  - **Cmd+Shift+Left / Cmd+Shift+Right** moves to the previous / next tab (Safari-style). Wraps around at the ends.
  - **Cmd+T** opens the folder picker to add a tab.
  - **Cmd+W** closes the active tab.
- **View mode**: Cmd+Opt+1 / 2 / 3 / 4 → Swimlanes / Timeline / List / Recent. (Cmd+1…9 belongs to tabs.)
- **Reload active tab**: Cmd+R.
- **Search** (related to #0007): Cmd+F focuses the search field.

## Notes

- Add a `CommandGroup`-based main menu so shortcuts route reliably (the `WindowGroup` today has none).
- ↑/↓ navigation needs a focus model — store and restore focus on view-mode switch so the user doesn't lose their place.
- For ↑/↓ to work cleanly when the detail panel is showing, the focus must live on the list, not the panel. `Table` already supports keyboard navigation natively when focused — verify this routes through `selection: $store.selectedIssueID` so the panel updates as the user scrolls. For Recent (custom rows), wire `.focusable()` + `.onKeyPress(.upArrow / .downArrow)`.
- Cmd+Shift+Left / Right wrap-around: `tabs[(idx - 1 + n) % n]` / `tabs[(idx + 1) % n]`. Disable wrap if there's only one tab (no-op).
- Cmd+1…9 maps `n -> tabs[n-1]`. If there are fewer than 9 tabs, the higher numbers no-op (don't error).
