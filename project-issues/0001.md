## Description

Clicking the `NNNN.md` link in the detail panel currently shells out to the user's default editor via `NSWorkspace.shared.open`. Replace that with an in-app modal that renders the markdown file using the new Textual Swift package (`gonzalezreal/textual`, already wired up in `Package.resolved`). The user should be able to read the issue's full markdown without leaving the app, then dismiss the modal back to the detail panel.

## Expected behavior

- Clicking the `NNNN.md` filename in the detail panel presents a sheet (or modal window) over the main window.
- The sheet renders the markdown file's contents formatted by Textual — headings, tables, lists, links, inline images, code blocks all styled.
- The sheet has a clear way to dismiss (close button and `Esc`).
- The modal updates if the underlying file changes while open (nice-to-have; the app already watches the folder).

## Actual behavior

The link calls `NSWorkspace.shared.open(issue.fileURL)` at `Issues/Views/DetailPanelView.swift:113`, which launches the system default editor for `.md` files. The user has to context-switch out of Issues.app to read the markdown.

## Notes

- Textual is already pinned in `Issues.xcodeproj/project.xcworkspace/xcshareddata/swiftpm/Package.resolved` (branch `main`).
- Entry point to change: the `Button { NSWorkspace.shared.open(...) }` block around `DetailPanelView.swift:113`.
- The metadata table at the top of each issue uses pipe-tables; confirm Textual renders those correctly. If not, the issue body still has prose worth showing.
- Inline images in `project-issues/NNNN/foo.png` use relative paths — the Textual view will need a base URL set to the issue file's parent directory so images resolve.
- Sandbox/security-scoped access: the app already holds a security-scoped bookmark for the issues folder. Reading the markdown inside the modal must happen while that scope is active.
