# Keyboard Shortcuts

Issues.app keeps shortcuts close to macOS conventions. The full set is below, grouped by what they do.

## Tabs

| Shortcut | Action |
|---|---|
| Cmd+T | New tab |
| Cmd+W | Close current tab |
| Cmd+R | Reload current tab |
| Cmd+1 … Cmd+9 | Jump to tab N |
| Cmd+Shift+Left | Previous tab |
| Cmd+Shift+Right | Next tab |

## View modes

| Shortcut | Action |
|---|---|
| Cmd+Opt+1 | Swimlanes view |
| Cmd+Opt+2 | Timeline view |
| Cmd+Opt+3 | List view |
| Cmd+Opt+4 | Recent view |

The view-mode shortcuts intentionally use Cmd+Opt so they don't fight the Cmd+1…9 tab bindings.

## Search and filter

| Shortcut | Action |
|---|---|
| Cmd+F | Focus the search field |
| Esc | Clear the search field when focused |

## Help

| Shortcut | Action |
|---|---|
| Cmd+? | Open this help window |

## A note on rebinding

These are the defaults. A future update will let you remap shortcuts from the Settings window; until then the values above are fixed in code. If a binding here clashes with a system-wide shortcut you've configured in System Settings → Keyboard → Keyboard Shortcuts, the system binding wins and the in-app one becomes unreachable.
